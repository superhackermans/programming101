Chapter 01 프로그래밍이란 실제로 무엇인가

핵심 개념

AI가 생성한 코드가 좋은지 나쁜지 판단하려면 먼저 코드가 무엇인지 알아야 합니다. 이 챕터는 커리큘럼 전체를 이해하는 데 필요한 핵심 사고 모델을 제공합니다.

프로그래밍(programming)이란 상식이 전혀 없는 기계에게 명령을 작성하는 일입니다. 누군가에게 땅콩버터 샌드위치를 만들어 달라고 한다고 상상해 보세요. 사람은 병을 집어 들고, 뚜껑을 열고, 칼을 집습니다. 컴퓨터는? "뚜껑을 열어라"라고 말하지 않았기 때문에 봉해진 병 앞에서 멈춰 버립니다. 모든 명령은 명시적이고, 문자 그대로이며, 순서가 있어야 합니다.

프로그램(program)은 이러한 명령들이 담긴 텍스트 파일입니다. 신비로운 바이너리 덩어리가 아니라, 어떤 편집기에서든 열 수 있는 일반 텍스트 파일입니다. 인터프리터(interpreter)(Python 같은 언어에서 사용)나 컴파일러(compiler)(Rust 같은 언어에서 사용)가 사람이 읽을 수 있는 텍스트를 기계의 프로세서가 실행할 수 있는 명령으로 번역합니다.

코드가 존재하는 이유

컴퓨터가 이해하는 것은 하나뿐입니다. 바로 전압 패턴입니다. 높은 전압 = 1, 낮은 전압 = 0. 이게 전부입니다. 프로그래밍 언어는 인간이 0과 1로 생각하지 않아도 되도록 존재합니다. 여러분이 작성하는 Python과 CPU에서 전환되는 전압 사이에는 여러 변환 계층이 있는데, 하위 계층은 건드릴 필요가 없습니다.

여러분은 Python을 작성합니다. 네 개의 변환 계층을 거쳐 CPU가 스위치를 전환합니다. 하위 계층은 건드릴 필요가 없습니다.

프로그래밍 언어의 세계

나사못에는 드라이버, 못에는 망치를 쓰듯이, 용도에 따라 서로 다른 프로그래밍 언어가 존재합니다. 주요 분류는 다음과 같습니다:

용도에 따라 다른 언어를 사용합니다. Python이 시작점입니다—AI, 스크립팅, 웹 백엔드를 모두 다룹니다.

소프트웨어는 실제로 어떻게 생겼나

위 다이어그램은 언어 분류를 보여 줍니다. 하지만 사람들이 그것으로 실제로 무엇을 만들까요? 거의 모든 소프트웨어는 몇 가지 주요 범주에 속하며, 대부분의 실제 제품은 여러 범주를 조합합니다.

웹 시스템 (프론트엔드 + 백엔드)

웹 애플리케이션(web application)은 브라우저에서 사용하는 모든 것을 말합니다. Gmail, YouTube, GitHub, 인터넷 뱅킹 등이 해당됩니다. 두 부분으로 나뉩니다:

  • 프론트엔드(frontend) — 눈에 보이는 부분입니다. HTML이 페이지 구조를 만들고, CSS가 스타일을 입히며, JavaScript가 인터랙션을 담당합니다. 버튼을 클릭했을 때 페이지를 새로 고침하지 않고 메뉴가 나타나는 것이 바로 브라우저에서 실행되는 JavaScript입니다.
  • 백엔드(backend) — 눈에 보이지 않는 부분입니다. 서버(주로 Python, Node.js, Go, Rust로 작성됨)가 요청을 처리하고, 비즈니스 규칙을 적용하며, 데이터베이스와 통신합니다. 로그인할 때 백엔드가 비밀번호를 확인하고, 댓글을 작성할 때 백엔드가 저장합니다.

프론트엔드와 백엔드는 HTTP를 통해 통신합니다—챕터 13에서 배울 바로 그 프로토콜(protocol)입니다. 방문하는 모든 URL은 어딘가의 백엔드에 HTTP 요청을 보냅니다.

실제 예시: Twitter/X의 경우, 프론트엔드가 브라우저에서 타임라인을 렌더링합니다. 백엔드는 인증을 처리하고, 트윗을 데이터베이스에 저장하며, 추천 알고리즘을 실행하고, JSON 형식으로 HTTP를 통해 타임라인 데이터를 제공합니다. 수백만 명의 사용자, 수천 개의 백엔드 서버, 하나의 끊김 없는 경험입니다.

모바일 애플리케이션

스마트폰의 앱입니다. 두 가지 접근 방식이 있습니다:

  • 네이티브 앱(native apps) — 특정 플랫폼 전용으로 개발됩니다. iOS는 Swift, Android는 Kotlin을 사용합니다. 최고의 성능과 기기 하드웨어(카메라, GPS, 센서)에 대한 완전한 접근을 제공합니다. Instagram, iOS 계산기, Apple 건강이 네이티브 앱입니다.
  • 크로스 플랫폼 앱(cross-platform apps) — 하나의 코드베이스로 두 플랫폼을 모두 지원합니다. React Native(JavaScript)나 Flutter(Dart)가 같은 코드에서 iOS와 Android 버전을 생성합니다. 개발 속도는 빠르지만 완성도가 약간 낮습니다. 많은 스타트업이 이 방식을 사용합니다.

대부분의 모바일 앱도 백엔드가 필요합니다—HTTP를 통해 같은 종류의 서버와 통신하는 다른 형태의 프론트엔드일 뿐입니다.

데이터베이스

데이터베이스(database)는 정리된 영구 저장소입니다. 데이터베이스가 없으면 프로그램이 종료될 때 데이터가 사라집니다. 두 가지 주요 계열이 있습니다:

  • 관계형(SQL) — 엄격한 규칙을 가진 스프레드시트처럼 행과 열로 이루어진 테이블에 데이터를 저장합니다. PostgreSQL, MySQL, SQLite가 해당됩니다. 데이터 간 관계가 명확할 때 적합합니다(사용자에게 주문이 있고, 주문에 아이템이 있는 경우). 챕터 15에서 SQLite를 사용합니다.
  • 비관계형(NoSQL) — 유연한 구조를 사용합니다. 문서(MongoDB), 키-값 쌍(Redis), 그래프(Neo4j) 등이 있습니다. 비정형 데이터, 캐싱, 또는 테이블에 깔끔하게 맞지 않는 관계에 적합합니다.

실제 예시: Shopify 같은 이커머스 사이트는 상품, 주문, 고객(구조화된 관계형 데이터)에 관계형 데이터베이스를 사용합니다. 상품 페이지가 즉시 로드되도록 캐싱(caching)에는 Redis(NoSQL)를, 상품 검색에는 Elasticsearch 같은 검색 엔진을 사용할 수 있습니다.

이들이 어떻게 함께 작동하는가

현대 애플리케이션은 일반적으로 세 가지를 모두 조합합니다. Uber 같은 차량 공유 앱을 생각해 보겠습니다:

  • 모바일 앱 (프론트엔드) — 지도를 보여 주고, 차량을 요청하고, 운전자 위치를 실시간으로 표시합니다
  • 백엔드 서버 — 승객과 운전자를 매칭하고, 요금을 계산하고, 결제를 처리하고, 수요 급증 요금 로직을 처리합니다
  • 데이터베이스 — 사용자 프로필, 이동 내역, 결제 수단, 운전자 평점을 저장합니다
  • 웹 대시보드 — 운전자가 계정을 관리하고, 관리자가 시스템을 모니터링할 수 있게 합니다

이 커리큘럼은 Python 백엔드와 SQLite 데이터베이스에 집중합니다. 이것이 토대이기 때문입니다. 백엔드가 요청을 처리하고 데이터를 저장하는 방식을 이해하고 나면, 프론트엔드, 모바일 앱, 클라우드 인프라 등 모든 것이 같은 패턴의 변형임을 알게 됩니다.

흔한 오해

주의: "수학을 잘해야 한다?"

프로그래밍은 방정식을 푸는 것보다 레시피를 쓰는 것에 더 가깝습니다. 대부분의 코드는 정보를 정리하고 프로세스를 기술하는 것이지, 미적분이 아닙니다. 수학이 많이 필요한 작업(머신 러닝, 물리 시뮬레이션)은 전문 분야이지 선행 조건이 아닙니다.

주의: "코드는 신비롭다?"

프로그램은 텍스트 파일입니다. 이 과정에서 만들 taskforge.py를 메모장으로 열어 모든 줄을 읽을 수 있습니다. 숨겨진 것은 없습니다. 나중에 AI가 코드를 생성할 때, 이것을 기억하면 중심을 잡을 수 있습니다—읽고 판단할 수 있는 텍스트일 뿐입니다.

3단계: 서버 시작

다음 명령을 입력하고 Enter를 누르세요:

터미널python3 -m http.server 8000

각 부분의 의미를 설명합니다. 여기서 마법처럼 느껴지는 것은 없어야 합니다:

  • python3 — 챕터 01에서 설치한 Python 프로그램을 실행합니다. 나중에 코드를 작성할 때 사용하는 것과 같은 Python입니다.
  • -m — 이 플래그는 Python에게 "이름으로 내장 모듈(Python에 미리 설치된 미니 프로그램)을 실행해"라고 지시합니다. 추가로 다운로드할 것이 없습니다. -m은 "모듈(module) 실행"이라고 생각하세요.
  • http.server — 내장 모듈의 이름입니다. 컴퓨터를 작은 웹 서버로 만듭니다. 서버는 요청을 기다렸다가 파일을 돌려보내는 프로그램—웹사이트를 방문할 때 일어나는 것과 같습니다. 차이점은 이 서버가 여러분의 컴퓨터에서 실행되며 여러분만 접근할 수 있다는 것입니다.
  • 8000 — 포트 번호(port number)입니다. 컴퓨터를 수천 개의 번호가 매겨진 문이 있는 건물로 생각하세요. 포트 8000은 서버가 듣도록 지시하는 문입니다. 8000이라는 숫자는 로컬 개발의 일반적인 관례입니다—거의 어떤 숫자도 될 수 있지만, 대부분의 튜토리얼이 8000을 사용하므로 이를 따르겠습니다.

표시되어야 하는 내용: Enter를 누르면 터미널에 다음과 같은 내용이 출력됩니다:

출력Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...

이는 서버가 실행 중이며 대기 중임을 의미합니다. 터미널이 "멈춘" 것처럼 보일 것입니다—정상입니다. 멈춘 것이 아니라 요청을 듣고 있는 것입니다. 서버가 실행되는 동안에는 해당 터미널 창에 더 이상 명령을 입력할 수 없습니다. (예상된 동작입니다. 나중에 Ctrl + C로 중지합니다.)

python3가 작동하지 않는 경우:

Chapter 02 컴퓨터가 코드를 실행하는 방법

핵심 개념

Claude Code가 함수를 생성하면, 머릿속으로 추적해야 합니다: 무엇이 들어가고, 무엇이 변하고, 무엇이 나오는지. 이 챕터는 컴퓨터가 어떻게 생각하는지—말 그대로 단계별로—알려줍니다.

프로그램이 명령이 담긴 텍스트 파일임을 알았으니, 기계가 실제로 그 명령을 어떻게 실행하는지 살펴봅시다. 핵심 개념: 순차 실행(sequential execution)—컴퓨터는 여러분이 작성한 순서대로 코드를 위에서 아래로, 한 번에 한 줄씩 읽습니다.

변수: 이름표가 붙은 상자

변수(variable)는 값을 담는 이름표가 붙은 상자입니다. 아파트 우편함 줄을 생각해 보세요. 각각 이름이 붙어 있고, 이름표를 바꾸지 않고도 안에 든 것을 교체할 수 있습니다. Python에서는 = 기호로 변수를 만듭니다:

# 이것은 Python입니다 — 이 가이드 전반에서 사용할 언어
speed = 85
name = "TaskForge"
running = True

이름표(speed)는 그대로 유지됩니다. 안에 든 값은 바꿀 수 있습니다. speed = 90은 같은 상자에 새 값을 넣습니다—이전 값 85는 사라집니다.

변수는 이름표가 붙은 상자입니다. 할당(assignment)은 상자에 새 값을 넣습니다. 이전 값은 사라집니다.

핵심 개념

모든 AI 코딩 도구—Claude Code, Cursor, Copilot—는 개발 환경에서 실행됩니다. 환경이 설정되지 않으면 이 커리큘럼에서 다른 것은 아무것도 작동하지 않습니다. 이 챕터가 여러분의 기계를 준비시킵니다.

Python 코드를 실행하려면 두 가지가 필요합니다: 터미널(명령을 입력하는 곳)과 Python(코드를 실행하는 인터프리터). 이 챕터에서 두 가지를 모두 준비합니다.

터미널

터미널은 컴퓨터에 대한 텍스트 기반 인터페이스입니다. 파일 → 새 폴더를 클릭하는 대신 mkdir new_folder를 입력합니다. 더 빠르고, 스크립트화할 수 있으며—결정적으로—모든 AI 코딩 도구가 작동하는 방식입니다. Claude Code는 완전히 터미널에서 실행됩니다.

터미널은 대화입니다. 명령을 입력하면 컴퓨터가 응답합니다. 프롬프트는 다음 명령을 받을 준비가 됐음을 알립니다.

터미널 에뮬레이터 업그레이드

핵심 개념

터미널 에뮬레이터(terminal emulator)는 셸(shell)에 접근하는 창입니다. 셸은 명령을 해석하는 프로그램(bash, zsh)이고, 터미널 에뮬레이터는 셸을 표시하고 입력을 처리하는 애플리케이션입니다. 모든 운영체제에 기본 터미널이 내장되어 있지만, 서드파티 터미널 에뮬레이터는 화면 분할, 검색, 즉시 재생 등 하루에 수 시간씩 터미널에서 작업할 때 큰 차이를 만드는 기능을 제공합니다.

Windows: Windows Terminal (Windows 11 기본 설치, Windows 10은 Microsoft Store에서 무료 다운로드). 탭, 화면 분할(Alt + Shift + D), 프로필을 지원합니다. 레거시 cmd.exe는 사용하지 마십시오.

가상 환경(Virtual Environment)

Python 패키지를 설치하기 전에 가상 환경(virtual environment)을 이해해야 합니다. 가상 환경은 특정 프로젝트를 위한 독립된 Python 사본입니다. 가상 환경 없이는 설치하는 모든 패키지가 하나의 전역 위치에 들어가게 되며—결국 두 프로젝트가 같은 패키지의 서로 다른 버전을 필요로 하게 됩니다. 그러면 문제가 생깁니다. 가상 환경은 이 문제를 완전히 방지합니다.

주의: pip install 전에 반드시 가상 환경을 만드십시오

가상 환경 밖에서 절대 pip install을 실행하지 마십시오. 전역으로 패키지를 설치하면 패키지가 쌓이고 충돌하며, 결국 진단하기 어려운 방식으로 무언가가 망가집니다. 가상 환경은 각 프로젝트의 패키지를 격리해 줍니다.

가상 환경 만들기

Terminal# Navigate to your project directory first cd ~/my-project # Create a virtual environment called .venv python3 -m venv .venv

이 명령은 프로젝트 안에 .venv 폴더를 만듭니다. 이 폴더에는 Python의 전용 사본과 깨끗하고 빈 패키지 집합이 들어 있습니다. 폴더 이름 .venv는 관례입니다—점(dot)으로 시작하면 숨김 폴더가 되어 파일 목록을 어지럽히지 않습니다.

핵심 개념

AI 코딩 도구 시대에는 작성하는 것보다 읽는 코드가 훨씬 많습니다. Claude Code가 200줄을 생성하면, 여러분의 일은 그것을 평가하는 것이지—직접 타이핑하는 것이 아닙니다. 이 챕터는 수동적인 수용자가 아닌, 유능한 AI 감독자가 되기 위한 스킬을 가르칩니다.

패러다임의 전환이 이루어졌습니다: 전통적인 교육은 코드를 작성하는 것에서 시작합니다. AI 지원 작업은 코드를 읽는 것에서 시작합니다. Claude Code가 200줄의 Python을 생성하면, 여러분은 그 줄들을 작성하는 것이 아니라—읽고, 평가하고, 수락할지 결정하게 됩니다. 코드 읽기가 여러분의 핵심 기술입니다.

코드 읽는 방법

1번째 줄부터 순서대로 읽지 마십시오. 다음과 같은 세 단계 접근법(three-pass approach)을 사용합니다:

1
구조 패스(Structure Pass)

함수 정의(def), 클래스, 임포트를 찾아봅니다. 구성 요소가 무엇인지 파악합니다.

2
로직 패스(Logic Pass)

함수 하나를 골라 데이터를 추적합니다: 무엇이 들어가고, 무엇이 나오고, 중간에 무엇이 변하는지 살펴봅니다.

3
엣지 패스(Edge Pass)

빈 입력이면? 거대한 입력이면? 잘못된 데이터면? 버그가 숨어 있는 곳이 바로 여기이며—AI 코드가 가장 자주 실패하는 곳이기도 합니다.

다음은 TaskForge v0.1의 전체 소스 코드입니다. 이것이 과정의 나머지 부분에서 쌓아 올릴 핵심 프로젝트(spine project)입니다. 세 단계 접근법을 사용하여 지금 읽어 보십시오.

# taskforge.py — v0.1
# A minimal command-line task manager

tasks = []

def add_task(description):
    task = {"id": len(tasks) + 1, "description": description, "status": "pending"}
    tasks.append(task)
    return task

def complete_task(task_id):
    for task in tasks:
        if task["id"] == task_id:
            task["status"] = "complete"
            return task
    return None

def list_tasks():
    return tasks

def main():
    while True:
        command = input("\nTaskForge> ").strip().lower()
        if command == "add":
            desc = input("Description: ")
            task = add_task(desc)
            print(f"Added: #{task['id']} — {task['description']}")
        elif command == "done":
            task_id = int(input("Task ID: "))
            result = complete_task(task_id)
            if result:
                print(f"Completed: #{result['id']}")
            else:
                print("Task not found.")
        elif command == "list":
            for t in list_tasks():
                status = "✓" if t["status"] == "complete" else " "
                print(f"  [{status}] #{t['id']} — {t['description']}")
        elif command == "quit":
            break
        else:
            print("Commands: add, done, list, quit")

if __name__ == "__main__":
    main()

    if len(numbers) == 0:
        return None

    return biggest
