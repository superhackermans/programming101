24장 탐색과 정렬

핵심 개념

모든 데이터베이스 쿼리는 탐색을 사용합니다. 모든 정렬된 화면 표시는 정렬을 사용합니다. AI는 정렬 코드를 끊임없이 생성합니다 — 올바른 알고리즘을 선택했는지 평가할 수 있어야 합니다. 잘못된 선택은 10밀리초의 응답과 10초의 응답 사이의 차이를 만들 수 있습니다. 이 장은 매번 그 판단을 내릴 수 있는 어휘와 이해력을 제공합니다.

선형 탐색 (Linear Search)

선형 탐색(linear search)은 가장 단순한 탐색 알고리즘입니다: 처음부터 시작하여 모든 원소를 확인하고, 목표값을 찾거나 원소가 다 떨어지면 멈춥니다. 시간 복잡도는 O(n)입니다 — 최악의 경우 모든 항목을 확인하게 됩니다.

def linear_search(arr, target):
    """Return the index of target in arr, or -1 if not found."""
    for i in range(len(arr)):      # Walk through every index
        if arr[i] == target:       # Found it?
            return i               # Return the position
    return -1                      # Exhausted the list without finding target

# Usage
tasks = ["buy groceries", "write tests", "deploy app", "review PR"]
idx = linear_search(tasks, "deploy app")
print(idx)  # 2

선형 탐색이 올바른 선택인 경우는 생각보다 많습니다:

  • 정렬되지 않은 데이터—정렬되지 않은 컬렉션에는 이진 탐색을 사용할 수 없습니다. 데이터가 임의 순서로 오고 한 번만 탐색한다면, 선형 탐색이 유일한 선택입니다.
  • 작은 컬렉션—원소가 50개 미만이면, 정렬된 구조를 유지하거나 해시 테이블을 구축하는 오버헤드가 단순 스캔 비용보다 큽니다. Python의 리스트 in 연산자는 내부적으로 선형 탐색을 사용합니다.
  • 일회성 탐색 시나리오—컬렉션을 정확히 한 번만 탐색한다면, 먼저 정렬(O(n log n)) 후 이진 탐색(O(log n))하는 것이 단일 선형 스캔(O(n))보다 느립니다.

Python의 리스트 in 연산자는 내부적으로 선형 탐색을 수행합니다:

tasks = ["buy groceries", "write tests", "deploy app"]
print("write tests" in tasks)  # True — scans left to right, O(n)

이진 탐색 (Binary Search)

이진 탐색(binary search)은 정렬된 데이터가 필요합니다. 탐색 공간을 반복적으로 절반으로 줄입니다: 목표값을 중간 원소와 비교한 후, 목표값을 포함할 수 없는 절반을 버립니다. 이로써 O(log n) 시간 복잡도를 달성합니다 — 백만 개의 원소 탐색이 최대 20번의 비교로 가능합니다.

알고리즘을 단계별로 살펴보면:

  1. lo = 0, hi = len(arr) - 1로 설정합니다.
  2. lo <= hi인 동안:
    1. mid = (lo + hi) // 2를 계산합니다.
    2. arr[mid] == target이면 mid를 반환합니다.
    3. arr[mid] < target이면 목표값은 오른쪽 절반에 있습니다: lo = mid + 1로 설정합니다.
    4. arr[mid] > target이면 목표값은 왼쪽 절반에 있습니다: hi = mid - 1로 설정합니다.
  3. 반환 없이 루프가 끝나면, 목표값이 배열에 없는 것입니다. -1을 반환합니다.
def binary_search(arr, target):
    """Return the index of target in sorted arr, or -1 if not found."""
    lo, hi = 0, len(arr) - 1           # 1. Initialize boundaries

    while lo <= hi:                     # 2. Loop while search space is non-empty
        mid = (lo + hi) // 2           #    Find the midpoint
        if arr[mid] == target:         #    2a. Exact match
            return mid
        elif arr[mid] < target:        #    2b. Target is in the right half
            lo = mid + 1               #         Discard left half including mid
        else:                          #    2c. Target is in the left half
            hi = mid - 1              #         Discard right half including mid

    return -1                          # 3. Not found

# Usage
sorted_ids = [2, 5, 8, 12, 16, 23, 38, 56, 72, 91]
print(binary_search(sorted_ids, 23))   # 5
print(binary_search(sorted_ids, 24))   # -1
주의

경계값 오류(off-by-one error)에 주의하십시오. 이진 탐색은 실수하기로 유명합니다. 가장 흔한 버그:

  • lo <= hi vs lo < hi—<= 대신 <를 사용하면 lo == hi인 경우를 건너뜁니다. 즉, 마지막 남은 원소를 절대 확인하지 않게 됩니다.
  • mid + 1 / mid - 1 vs mid—lo = mid + 1 대신 lo = mid로 설정하면 lo == mid일 때 (hi - lo == 1일 때 발생) 무한 루프가 생길 수 있습니다.
  • 정수 오버플로(integer overflow)—C++이나 Java 같은 언어에서는 (lo + hi)가 오버플로될 수 있습니다. Python은 임의 정밀도 정수를 처리하므로 Python에서는 문제가 없지만, 다른 언어로 작업할 때는 주의하십시오.

탐색 공간에 대한 이진 탐색 (Binary Search on a Search Space)

이진 탐색은 배열에서 원소를 찾는 것에만 국한되지 않습니다. "X가 충분히 큰가?"에 예/아니오로 답할 수 있는 모든 문제는 답의 공간에 대해 이진 탐색으로 풀 수 있습니다. 이것은 가장 강력한 알고리즘 기법 중 하나이며, 실제 엔지니어링에서 끊임없이 등장합니다.

문제: 작업 소요 시간 목록과 작업자 수가 주어집니다. 각 작업자는 작업을 순차적으로 처리합니다. 어떤 단일 작업자가 소요하는 최대 시간이 최소화되도록 작업을 작업자들에게 분배하고 싶습니다. 그 최소 최대 시간은 얼마입니까?

핵심 아이디어: 시간 제한 T를 추측하면, 탐욕적으로(greedily) T가 실현 가능한지 확인할 수 있습니다 — 현재 작업자가 T를 초과할 때마다 새 작업자를 시작하여 작업을 순서대로 할당합니다. 가용 작업자보다 더 많이 필요하다면 T는 너무 작은 것입니다. 더 적거나 정확히 맞는다면, T는 실현 가능합니다. 정답은 가장 작은 실현 가능한 T입니다.

def min_max_time(durations, num_workers):
    """Find the minimum possible maximum time per worker."""

    def is_feasible(max_time):
        """Can we assign all tasks using <= num_workers if each handles <= max_time?"""
        workers_needed = 1
        current_load = 0
        for d in durations:
            if current_load + d > max_time:
                workers_needed += 1        # Start a new worker
                current_load = d           # This task goes to the new worker
            else:
                current_load += d          # Add task to current worker
        return workers_needed <= num_workers

    lo = max(durations)          # Lower bound: at least the longest single task
    hi = sum(durations)          # Upper bound: one worker does everything

    while lo < hi:
        mid = (lo + hi) // 2
        if is_feasible(mid):
            hi = mid             # mid works — try to find something smaller
        else:
            lo = mid + 1         # mid is too small — need more time

    return lo

# Example: 5 tasks, 3 workers
print(min_max_time([10, 20, 30, 40, 50], 3))  # 60

원소 탐색 이진 탐색과의 미묘한 차이점을 주목하십시오: 여기서는 lo < hi를 사용하고 (<=가 아님), lo와 hi가 같은 값으로 수렴합니다. 이것이 표준적인 "최댓값 최소화" 이진 탐색 템플릿입니다.

삽입 정렬 (Insertion Sort)

삽입 정렬(insertion sort)은 배열의 정렬된 부분을 한 번에 한 원소씩 구축합니다. 각 새 원소에 대해, 큰 원소들을 오른쪽으로 이동시켜 공간을 만든 후, 올바른 위치에 원소를 삽입합니다.

def insertion_sort(arr):
    """Sort arr in-place using insertion sort."""
    for i in range(1, len(arr)):       # Start from the second element
        key = arr[i]                   # The element to insert
        j = i - 1                      # Start comparing with the element to its left

        # Shift elements that are larger than key one position to the right
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]        # Shift right
            j -= 1                     # Move comparison pointer left

        arr[j + 1] = key               # Insert key in the correct position

    return arr

print(insertion_sort([5, 2, 8, 1, 9, 3]))  # [1, 2, 3, 5, 8, 9]

복잡도: 최악의 경우(역순 정렬 입력) O(n²)이지만, 데이터가 거의 정렬되어 있을 때 최선의 경우 O(n)입니다. 이로 인해 삽입 정렬은 작은 배열이나 "거의 정렬된" 배열에 탁월합니다 — 실제로는 놀랍도록 흔한 경우입니다. Python의 Timsort도 작은 실행(run)에는 삽입 정렬을 사용합니다.

병합 정렬 (Merge Sort)

병합 정렬(merge sort)은 분할 정복(divide-and-conquer) 알고리즘입니다. 각 조각이 하나의 원소가 될 때까지(이미 정렬된 상태) 배열을 재귀적으로 절반으로 분할한 후, 정렬된 절반들을 다시 합칩니다. 입력 순서에 관계없이 O(n log n) 시간을 보장하며, 안정적(stable)입니다 — 동등한 원소들은 원래 상대적 순서를 유지합니다.

def merge_sort(arr):
    """Return a new sorted list using merge sort."""
    if len(arr) <= 1:                  # Base case: 0 or 1 elements are already sorted
        return arr

    mid = len(arr) // 2                # Find the midpoint
    left = merge_sort(arr[:mid])       # Recursively sort the left half
    right = merge_sort(arr[mid:])      # Recursively sort the right half

    return merge(left, right)          # Merge the two sorted halves

def merge(left, right):
    """Merge two sorted lists into one sorted list."""
    result = []
    i = j = 0                         # Pointers into left and right

    while i < len(left) and j < len(right):
        if left[i] <= right[j]:        # <= preserves stability
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1

    # One of the halves may have remaining elements
    result.extend(left[i:])
    result.extend(right[j:])
    return result

print(merge_sort([38, 27, 43, 3, 9, 82, 10]))  # [3, 9, 10, 27, 38, 43, 82]

재귀 트리: 8개 원소 배열이 두 개의 4원소 배열로 분할되고, 이것이 네 개의 2원소 배열로, 다시 여덟 개의 1원소 배열로 분할됩니다. 분할 레벨이 log₂ n개입니다. 각 레벨에서 병합 단계가 모든 원소를 한 번씩 처리하므로, 각 레벨의 비용은 O(n)입니다. 합계: O(n log n).

공간 복잡도: O(n) — 병합 함수가 각 레벨에서 새 목록을 생성합니다. 이것이 퀵 정렬과 같은 제자리 정렬(in-place sort)과의 트레이드오프입니다.

병합 정렬은 배열을 재귀적으로 분할하고(위쪽 절반), 정렬된 부분 배열을 다시 합칩니다(아래쪽 절반). 각 병합 레벨은 모든 원소를 한 번씩 처리하여 총 O(n log n)의 작업량이 됩니다.

퀵 정렬 (Quicksort)

퀵 정렬(quicksort)은 또 다른 분할 정복 알고리즘이지만, 중간점에서 분할하는 대신 피벗(pivot) 원소를 기준으로 배열을 분할(partition)합니다. 피벗보다 작은 원소는 모두 왼쪽으로, 큰 원소는 모두 오른쪽으로 갑니다. 그런 다음 왼쪽과 오른쪽 파티션을 재귀적으로 정렬합니다.

def quicksort(arr):
    """Return a new sorted list using quicksort."""
    if len(arr) <= 1:
        return arr

    pivot = arr[len(arr) // 2]                # Choose middle element as pivot
    left = [x for x in arr if x < pivot]      # Elements smaller than pivot
    middle = [x for x in arr if x == pivot]    # Elements equal to pivot
    right = [x for x in arr if x > pivot]     # Elements larger than pivot

    return quicksort(left) + middle + quicksort(right)

print(quicksort([10, 7, 8, 9, 1, 5]))  # [1, 5, 7, 8, 9, 10]

이것은 "Lomuto 방식" 리스트 컴프리헨션 버전입니다 — 명확하지만 추가 공간을 사용합니다. 제자리(in-place) Hoare 분할 방식은 메모리 효율이 더 높지만 읽기가 어렵습니다:

def quicksort_inplace(arr, lo=0, hi=None):
    """Sort arr in-place using quicksort with Hoare partition."""
    if hi is None:
        hi = len(arr) - 1
    if lo >= hi:
        return

    # Partition
    pivot = arr[(lo + hi) // 2]
    i, j = lo, hi
    while i <= j:
        while arr[i] < pivot:
            i += 1
        while arr[j] > pivot:
            j -= 1
        if i <= j:
            arr[i], arr[j] = arr[j], arr[i]
            i += 1
            j -= 1

    quicksort_inplace(arr, lo, j)
    quicksort_inplace(arr, i, hi)

data = [10, 7, 8, 9, 1, 5]
quicksort_inplace(data)
print(data)  # [1, 5, 7, 8, 9, 10]

복잡도: 평균적으로 O(n log n)이지만, 최악의 경우 O(n²)입니다 — 피벗이 항상 가장 작거나 가장 큰 원소일 때 발생합니다(예: 첫 번째 원소를 피벗으로 사용할 때 이미 정렬된 입력). 이를 방지하는 피벗 선택 전략:

  • 무작위 피벗—무작위 원소를 선택합니다. 최악의 경우가 천문학적으로 희박해집니다.
  • 중앙값-셋(Median-of-three)—첫 번째, 중간, 마지막 원소의 중앙값을 선택합니다. 정렬된 입력을 잘 처리합니다.
  • 중간 원소—위에서 사용한 방식. 단순하고 대체로 효과적입니다.

힙 정렬 (Heapsort)

힙 정렬(heapsort)은 배열에서 최대 힙(max-heap)을 구축한 후(22장 참조), 최대 원소를 반복적으로 추출하여 끝에 배치합니다. O(n log n)의 보장된 시간을 달성하며, 제자리(in-place)입니다 — 추가 배열이 필요 없습니다.

def heapsort(arr):
    """Sort arr in-place using heapsort."""
    n = len(arr)

    def sift_down(start, end):
        """Restore the max-heap property from start down to end."""
        root = start
        while 2 * root + 1 <= end:
            child = 2 * root + 1        # Left child
            if child + 1 <= end and arr[child] < arr[child + 1]:
                child += 1              # Right child is larger
            if arr[root] < arr[child]:
                arr[root], arr[child] = arr[child], arr[root]
                root = child
            else:
                return

    # Phase 1: Build a max-heap (heapify)
    for i in range(n // 2 - 1, -1, -1):
        sift_down(i, n - 1)

    # Phase 2: Extract max repeatedly
    for end in range(n - 1, 0, -1):
        arr[0], arr[end] = arr[end], arr[0]   # Move max to sorted position
        sift_down(0, end - 1)                  # Restore heap on reduced array

    return arr

data = [4, 10, 3, 5, 1]
print(heapsort(data))  # [1, 3, 4, 5, 10]

힙 정렬은 추가 메모리 없이 보장된 O(n log n)이 필요할 때 유용합니다. 실제로는 더 나은 캐시 지역성(cache locality) 때문에 퀵 정렬이 더 빠른 경우가 많지만, 힙 정렬의 최악의 경우는 엄격히 O(n log n)입니다 — 불운한 피벗이 없습니다.

비교 기반이 아닌 정렬 (Non-Comparison Sorts)

위의 모든 정렬들은 원소 쌍을 비교합니다. 비교 하한(comparison lower bound)이라 불리는 수학적 결과는 어떤 비교 기반 정렬도 O(n log n)보다 나을 수 없다는 것을 증명합니다. 하지만 데이터가 알려진 범위 내의 정수로 구성된 경우 이 장벽을 깰 수 있습니다.

계수 정렬(counting sort)은 각 값의 발생 횟수를 세고, 계수에서 정렬된 배열을 재구성합니다. 시간: O(n + k), 여기서 k는 값의 범위입니다.

def counting_sort(arr, max_val):
    """Sort non-negative integers using counting sort. O(n + k)."""
    counts = [0] * (max_val + 1)        # One counter per possible value
    for x in arr:
        counts[x] += 1                  # Count occurrences

    result = []
    for val, count in enumerate(counts):
        result.extend([val] * count)    # Emit each value 'count' times
    return result

print(counting_sort([4, 2, 2, 8, 3, 3, 1], 8))  # [1, 2, 2, 3, 3, 4, 8]

기수 정렬(radix sort)은 최하위 자릿수부터 최상위 자릿수까지 각 자릿수별로 정수를 정렬하며, 각 자릿수에서 안정적인 정렬(계수 정렬 등)을 사용합니다. 시간: O(d × (n + k)), 여기서 d는 자릿수, k는 기수(보통 10)입니다. 고정 크기 정수의 경우 사실상 O(n)입니다.

안정성 (Stability)

동등한 키를 가진 원소들이 원래 상대적 순서를 유지하면 정렬이 안정적(stable)이라고 합니다. 이것은 여러 키로 정렬할 때 중요합니다.

예시: 생성 날짜로 정렬된 작업들이 있습니다. 이제 우선순위로 정렬합니다. 안정적인 정렬을 사용하면, 같은 우선순위의 작업들은 생성 날짜 순서를 유지합니다. 불안정한 정렬을 사용하면, 우선순위 레벨 내에서의 순서를 예측할 수 없습니다.

Timsort: Python이 실제로 사용하는 것

Python에서 sorted()나 list.sort()를 호출하면 Timsort를 사용합니다 — 2002년 Tim Peters가 Python을 위해 특별히 발명한 하이브리드 알고리즘입니다. Timsort는 병합 정렬과 삽입 정렬을 결합하며, 실제 데이터에서 나타나는 패턴을 활용하도록 설계되었습니다:

  • 입력에서 기존의 정렬된 "실행(run)"(오름차순 또는 내림차순 시퀀스)을 찾습니다.
  • 짧은 실행은 삽입 정렬을 사용하여 최소 길이로 확장됩니다.
  • 실행들은 불균형 병합을 위한 "갤로핑(galloping)" 모드가 포함된 수정된 병합 정렬로 쌍씩 병합됩니다.

Timsort는 최악의 경우 O(n log n), 이미 정렬된 데이터에서 O(n)이며, 안정적입니다. 매우 효과적이어서 Java, Rust, Swift도 표준 라이브러리에 채택했습니다.

실용적인 결론: Python의 내장 sorted()를 사용하십시오. C로 구현되어 있고 Timsort의 실제 세계 최적화를 사용하기 때문에, 직접 작성한 것보다 거의 확실히 빠릅니다. 학습 목적이나 제약된 환경에서만 직접 정렬을 구현하십시오.

25장 재귀와 동적 프로그래밍

핵심 개념

많은 문제들은 우아한 재귀적 해법을 가지고 있습니다. 하지만 순진한 재귀는 천문학적으로 느릴 수 있습니다 — Fibonacci(50)을 재귀적으로 계산하면 1조 번 이상의 함수 호출이 발생합니다. 동적 프로그래밍(dynamic programming)은 이러한 지수적 알고리즘을 다항식 알고리즘으로 변환합니다. AI는 재귀 코드를 즐겨 생성합니다. 언제 효율적이고 언제 프로덕션 시스템을 충돌시키는 함정인지 알아야 합니다.

재귀적 사고 (Recursive Thinking)

재귀 함수(recursive function)는 같은 문제의 더 작은 버전으로 자신을 호출합니다. 모든 재귀에는 두 부분이 필요합니다:

  1. 기저 사례(base case)—재귀를 멈추는 조건. 이것이 없으면 함수가 영원히 자신을 호출합니다.
  2. 재귀 사례(recursive case)—기저 사례를 향해 나아가는 더 작은 입력으로 자신을 호출합니다.

가장 단순한 예시는 팩토리얼입니다:

def factorial(n):
    """Compute n! = n * (n-1) * ... * 1."""
    if n <= 1:             # Base case: 0! = 1! = 1
        return 1
    return n * factorial(n - 1)  # Recursive case: n! = n * (n-1)!

print(factorial(5))  # 120 = 5 * 4 * 3 * 2 * 1

실행 과정을 살펴보면:

factorial(5)
  = 5 * factorial(4)
  = 5 * (4 * factorial(3))
  = 5 * (4 * (3 * factorial(2)))
  = 5 * (4 * (3 * (2 * factorial(1))))
  = 5 * (4 * (3 * (2 * 1)))           # Base case reached
  = 5 * (4 * (3 * 2))
  = 5 * (4 * 6)
  = 5 * 24
  = 120

이제 피보나치(Fibonacci) — 각 숫자는 앞의 두 숫자의 합입니다: 0, 1, 1, 2, 3, 5, 8, 13, 21, ...

def fib(n):
    """Return the nth Fibonacci number (naive recursive)."""
    if n <= 1:             # Base cases: fib(0) = 0, fib(1) = 1
        return n
    return fib(n - 1) + fib(n - 2)  # Recursive case

print(fib(10))  # 55

이것은 작동하지만, 큰 n에 대해서는 치명적으로 느립니다. 왜냐하면 같은 부분 문제를 반복해서 재계산하기 때문입니다. fib(5)는 fib(3)을 두 번, fib(2)를 세 번, fib(1)을 다섯 번 호출합니다. 총 호출 횟수는 지수적으로 증가합니다: O(2ⁿ).

호출 스택 (Call Stack)

각 재귀 호출은 함수의 지역 변수와 반환 주소를 저장하는 메모리 블록인 스택 프레임(stack frame)을 추가합니다. 프레임들은 재귀가 깊어질수록 쌓이고, 각 호출이 반환됨에 따라 해제됩니다.

# Visualizing the call stack for factorial(4):
#
# Frame 4: factorial(1) -> returns 1         [TOP]
# Frame 3: factorial(2) -> waiting for frame 4
# Frame 2: factorial(3) -> waiting for frame 3
# Frame 1: factorial(4) -> waiting for frame 2  [BOTTOM]
#
# After factorial(1) returns, frames unwind from top to bottom.

Python은 기본 재귀 한도(recursion limit)를 1000으로 설정합니다. 이를 확인하고 변경할 수 있습니다:

import sys
print(sys.getrecursionlimit())  # 1000
sys.setrecursionlimit(5000)     # Increase it
주의: setrecursionlimit은 보통 잘못된 해결책입니다

재귀가 한도에 도달하면, 실제 문제는 대개 한도가 너무 낮은 것이 아니라 알고리즘의 깊이가 지수적이라는 것입니다. 한도를 높이면 충돌을 늦출 뿐이며 사용 가능한 모든 메모리를 소비할 위험이 있습니다. 올바른 해결책은 반복으로 변환하거나 메모이제이션을 추가하는 것입니다.

스택 오버플로 (Stack Overflow)

재귀가 너무 깊어지면 RecursionError가 발생합니다 (Python의 스택 오버플로 버전). 이것은 다음과 같은 경우에 발생합니다:

  • 기저 사례를 잊어버린 경우
  • 기저 사례가 잘못되어 절대 트리거되지 않는 경우
  • 문제가 재귀 깊이에 비해 너무 큰 경우 (예: 100,000개 원소의 연결 리스트를 재귀적으로 처리)

꼬리 재귀(tail recursion)는 재귀 호출이 함수의 맨 마지막 연산인 패턴입니다. 일부 언어(Scheme, Scala)는 이것을 자동으로 루프로 최적화하여 스택 성장을 제거합니다. Python은 꼬리 호출 최적화(tail-call optimization)를 하지 않습니다. Python에서 깊은 재귀가 필요하면, 스택 자료 구조를 사용하는 명시적 루프로 변환하십시오.

# Recursive (limited by call stack):
def sum_list_recursive(lst, i=0):
    if i == len(lst):
        return 0
    return lst[i] + sum_list_recursive(lst, i + 1)

# Iterative (no stack limit):
def sum_list_iterative(lst):
    total = 0
    for x in lst:
        total += x
    return total

메모이제이션 (Memoization)

메모이제이션(memoization)은 비용이 많이 드는 함수 호출의 결과를 캐시하여, 같은 인수로 함수가 다시 호출되면 재계산 대신 캐시된 결과를 반환합니다. 이로써 O(2ⁿ) 피보나치가 O(n)으로 변환됩니다.

def fib_memo(n, cache={}):
    """Fibonacci with manual memoization."""
    if n in cache:             # Already computed? Return cached result.
        return cache[n]
    if n <= 1:
        return n
    cache[n] = fib_memo(n - 1, cache) + fib_memo(n - 2, cache)
    return cache[n]

print(fib_memo(50))  # 12586269025 — instant, not heat-death-of-the-universe

Python은 이것을 자동으로 수행하는 내장 데코레이터를 제공합니다:

from functools import lru_cache

@lru_cache(maxsize=None)    # Cache all results (no eviction)
def fib_cached(n):
    if n <= 1:
        return n
    return fib_cached(n - 1) + fib_cached(n - 2)

print(fib_cached(100))  # 354224848179261915075

차이는 극적입니다. 시간을 측정해 봅시다:

import time

# Naive recursive — try n=35 (anything higher may freeze your machine)
start = time.time()
fib(35)
print(f"Naive: {time.time() - start:.2f}s")  # ~3-5 seconds

# Memoized — same n but now try n=500

print(f"Memoized: {time.time() - start:.6f}s")  # ~0.000 seconds

상향식 표 작성 (Bottom-Up Tabulation)

메모이제이션은 "하향식(top-down)"입니다 — 큰 문제에서 시작하여 아래로 재귀합니다. 표 작성(tabulation)은 "상향식(bottom-up)"입니다 — 기저 사례부터 시작하여 위쪽으로 표를 구축합니다. 재귀도, 스택 오버헤드도 없습니다.

def fib_table(n):
    """Fibonacci using bottom-up tabulation."""
    if n <= 1:
        return n

    table = [0] * (n + 1)     # table[i] will hold fib(i)
    table[0] = 0               # Base case
    table[1] = 1               # Base case

    for i in range(2, n + 1):
        table[i] = table[i - 1] + table[i - 2]   # Build from smaller subproblems

    return table[n]

print(fib_table(100))  # 354224848179261915075

이전 두 값만 필요하므로 공간을 O(1)로 최적화할 수도 있습니다:

def fib_optimal(n):
    """Fibonacci with O(1) space."""
    if n <= 1:
        return n
    prev2, prev1 = 0, 1
    for _ in range(2, n + 1):
        prev2, prev1 = prev1, prev2 + prev1
    return prev1

print(fib_optimal(100))  # 354224848179261915075

고전적인 동적 프로그래밍 패턴

동전 거스름돈: 목표 금액을 만들기 위한 최소 동전 수

문제: 동전 액면가(예: [1, 5, 10, 25])와 목표 금액이 주어졌을 때, 필요한 최소 동전 수를 찾으십시오.

이것은 표준적인 DP 문제입니다. 핵심 통찰: 금액 a에 대한 최소 동전 수는 모든 동전 c에 대해 금액 a - c의 답에 1을 더한 것의 최솟값입니다.

def coin_change(coins, amount):
    """Return minimum coins to make amount, or -1 if impossible."""
    # dp[i] = minimum coins needed to make amount i
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0                          # Base case: 0 coins for amount 0

    for a in range(1, amount + 1):     # For each amount from 1 to target
        for c in coins:                # Try each coin denomination
            if c <= a:                 # Coin must not exceed the current amount
                dp[a] = min(dp[a], dp[a - c] + 1)

    return dp[amount] if dp[amount] != float('inf') else -1

print(coin_change([1, 5, 10, 25], 36))  # 3 (25 + 10 + 1)
print(coin_change([3, 7], 5))            # -1 (impossible)

복잡도: O(amount × len(coins)). 표의 각 셀은 동전당 일정한 작업량으로 한 번만 계산됩니다.

최장 공통 부분 수열 (Longest Common Subsequence, LCS)

문제: 두 문자열이 주어졌을 때, 최장 공통 부분 수열(LCS)의 길이를 찾으십시오 — 두 문자열에서 같은 순서로 나타나는(반드시 연속적일 필요는 없는) 가장 긴 문자 시퀀스입니다.

이것은 diff 알고리즘(git diff 등)의 기반입니다: 두 파일의 LCS를 찾고, LCS에 없는 모든 것이 변경 사항입니다.

def lcs(s1, s2):
    """Return the length of the longest common subsequence."""
    m, n = len(s1), len(s2)
    # dp[i][j] = LCS length of s1[:i] and s2[:j]
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i - 1] == s2[j - 1]:          # Characters match
                dp[i][j] = dp[i - 1][j - 1] + 1  # Extend the LCS
            else:                                  # Characters differ
                dp[i][j] = max(dp[i - 1][j],      # Skip char from s1
                               dp[i][j - 1])      # Skip char from s2

    return dp[m][n]

print(lcs("ABCBDAB", "BDCAB"))  # 4 ("BCAB")

0/1 배낭 문제 (0/1 Knapsack): 무게 제한 내에서 가치 최대화

문제: 무게와 가치가 있는 아이템들이 있습니다. 무게 용량이 있는 배낭이 있습니다. 용량을 초과하지 않으면서 총 가치를 최대화하십시오. 각 아이템은 최대 한 번만 가져갈 수 있습니다.

def knapsack(values, weights, capacity):
    """Return maximum value achievable within capacity."""
    n = len(values)
    # dp[i][w] = max value using items 0..i-1 with capacity w
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        for w in range(capacity + 1):
            dp[i][w] = dp[i - 1][w]               # Don't take item i
            if weights[i - 1] <= w:                 # Can we take item i?
                take = dp[i - 1][w - weights[i - 1]] + values[i - 1]
                dp[i][w] = max(dp[i][w], take)     # Take it if better

    return dp[n][capacity]

# Items: (value, weight)
values  = [60, 100, 120]
weights = [10,  20,  30]
capacity = 50
print(knapsack(values, weights, capacity))  # 220 (items 2 and 3)

복잡도: O(n × capacity). 이것은 의사 다항식(pseudo-polynomial)이라 불립니다 — 입력 크기뿐만 아니라 capacity의 수치에도 의존합니다.

동적 프로그래밍 문제 인식하기

모든 문제가 DP의 혜택을 받는 것은 아닙니다. 두 가지 속성이 충족되어야 합니다:

  1. 중복 부분 문제(overlapping subproblems)—같은 부분 문제가 여러 번 풀립니다. 모든 부분 문제가 고유하면(병합 정렬처럼), DP는 이점 없이 오버헤드만 추가합니다.
  2. 최적 부분 구조(optimal substructure)—전체 문제의 최적 해는 부분 문제의 최적 해로 구축할 수 있습니다. 부분 결정을 변경해도 다른 부분 결정에 영향을 주지 않으면, 문제에 최적 부분 구조가 있습니다.

빠른 테스트: 재귀적 해법을 작성할 수 있습니까? 같은 부분 문제를 재계산합니까? 둘 다 예라면 메모이제이션을 추가하십시오. 부분 문제 표의 차원을 파악할 수 있다면, 상향식 버전을 작성하십시오.

왼쪽: 순진한 재귀 피보나치는 부분 문제를 지수적으로 재계산합니다. 오른쪽: 메모이제이션 버전은 결과를 캐시하여(음영 처리된 상자), O(n)의 고유한 계산으로 줄입니다.

TaskForge 연결

TaskForge: 최적 작업 선택 (배낭 문제)

각각 시간 비용과 가치(우선순위 점수)가 있는 TaskForge 작업 집합이 있습니다. 스프린트에 대한 시간 예산이 있습니다. 시간 예산 내에서 총 가치를 최대화하려면 어떤 작업을 선택해야 합니까? 이것이 0/1 배낭 문제입니다.

def select_tasks(tasks, time_budget):
    """Select tasks to maximize total value within time_budget.
    Each task is a dict with 'title', 'time', and 'value' keys.
    Returns list of selected task titles.
    """
    n = len(tasks)
    dp = [[0] * (time_budget + 1) for _ in range(n + 1)]

    for i in range(1, n + 1):
        t = tasks[i - 1]
        for w in range(time_budget + 1):
            dp[i][w] = dp[i - 1][w]              # Don't take this task
            if t["time"] <= w:
                take = dp[i - 1][w - t["time"]] + t["value"]
                dp[i][w] = max(dp[i][w], take)

    # Backtrack to find which tasks were selected
    selected = []
    w = time_budget
    for i in range(n, 0, -1):
        if dp[i][w] != dp[i - 1][w]:
            selected.append(tasks[i - 1]["title"])
            w -= tasks[i - 1]["time"]

    return selected

sprint_tasks = [
    {"title": "Auth system",     "time": 8,  "value": 90},
    {"title": "Search feature",  "time": 5,  "value": 60},
    {"title": "Fix CSS bug",     "time": 1,  "value": 20},
    {"title": "API pagination",  "time": 3,  "value": 45},
    {"title": "Dark mode",       "time": 4,  "value": 30},
]
print(select_tasks(sprint_tasks, 10))
# ['API pagination', 'Search feature', 'Fix CSS bug'] — value 125 in 9 hours

26장 알고리즘 패턴

핵심 개념

대부분의 알고리즘 문제는 소수의 패턴에 맞아들어갑니다. 패턴을 인식하는 것이 해법의 80%입니다. 이것들은 단순한 인터뷰 트릭이 아닙니다 — 실제 엔지니어링 솔루션의 구성 요소입니다. 속도 제한기(rate limiter)는 슬라이딩 윈도우를 사용합니다. 스케줄러는 탐욕 알고리즘을 사용합니다. 구성 검증기는 백트래킹을 사용합니다. 패턴을 한번 보면, 어디서나 보이게 됩니다.

투 포인터 (Two Pointers)

투 포인터(two-pointer) 기법은 배열을 이동하는 두 개의 인덱스를 사용합니다. 반대 끝에서 서로를 향해 이동하거나 같은 방향으로 다른 속도로 이동합니다. 많은 O(n²) 브루트 포스 해법을 O(n)으로 전환합니다.

패턴 1: 수렴하는 포인터 (반대 끝)

문제: 정렬된 배열과 목표 합이 주어졌을 때, 목표에 더해지는 원소 쌍을 찾으십시오.

def two_sum_sorted(arr, target):
    """Find two numbers in sorted arr that sum to target. Return their indices."""
    left, right = 0, len(arr) - 1

    while left < right:
        current_sum = arr[left] + arr[right]
        if current_sum == target:
            return (left, right)
        elif current_sum < target:
            left += 1          # Need a bigger sum — move left pointer right
        else:
            right -= 1         # Need a smaller sum — move right pointer left

    return None                # No pair found

print(two_sum_sorted([1, 3, 5, 7, 11, 15], 16))  # (2, 4) — 5 + 11

작동 원리: 배열이 정렬되어 있으므로, 왼쪽 포인터를 오른쪽으로 이동하면 합이 증가하고, 오른쪽 포인터를 왼쪽으로 이동하면 합이 감소합니다. 이미 배제한 쌍을 다시 확인할 필요가 없습니다.

패턴 2: 팰린드롬 확인

def is_palindrome(s):
    """Check if string is a palindrome, ignoring case and non-alphanumeric chars."""
    left, right = 0, len(s) - 1

    while left < right:
        while left < right and not s[left].isalnum():
            left += 1
        while left < right and not s[right].isalnum():
            right -= 1
        if s[left].lower() != s[right].lower():
            return False
        left += 1
        right -= 1

    return True

print(is_palindrome("A man, a plan, a canal: Panama"))  # True

패턴 3: 제자리 중복 제거

def remove_duplicates(arr):
    """Remove duplicates from sorted arr in-place. Return new length."""
    if not arr:
        return 0

    write = 1                          # Position to write next unique element
    for read in range(1, len(arr)):    # Scan through the array
        if arr[read] != arr[read - 1]: # Found a new value
            arr[write] = arr[read]     # Write it
            write += 1

    return write

data = [1, 1, 2, 2, 2, 3, 4, 4, 5]
new_len = remove_duplicates(data)
print(data[:new_len])  # [1, 2, 3, 4, 5]

슬라이딩 윈도우 (Sliding Window)

슬라이딩 윈도우(sliding window) 기법은 "윈도우"(연속적인 부분 배열 또는 부분 문자열)를 유지하고 데이터를 가로질러 슬라이드합니다. 두 가지 유형이 있습니다:

고정 크기 윈도우: k개 연속 원소의 최대 합

def max_sum_k(arr, k):
    """Return the maximum sum of k consecutive elements."""
    if len(arr) < k:
        return None

    # Compute the sum of the first window
    window_sum = sum(arr[:k])
    max_sum = window_sum

    # Slide: add the next element, remove the leftmost element
    for i in range(k, len(arr)):
        window_sum += arr[i] - arr[i - k]   # Slide the window
        max_sum = max(max_sum, window_sum)

    return max_sum

print(max_sum_k([2, 1, 5, 1, 3, 2], 3))  # 9 (5 + 1 + 3)

각 위치에서 k개 원소의 합을 재계산하는 대신(O(n × k)), 한 원소씩 조정하여(O(1)) 총 O(n)을 달성합니다.

가변 크기 윈도우: 반복 문자가 없는 가장 긴 부분 문자열

def longest_unique_substring(s):
    """Return the length of the longest substring without repeating characters."""
    seen = {}                  # Map character -> its most recent index
    max_len = 0
    start = 0                  # Left edge of the window

    for end in range(len(s)):  # Right edge expands
        if s[end] in seen and seen[s[end]] >= start:
            start = seen[s[end]] + 1   # Shrink: jump past the duplicate
        seen[s[end]] = end
        max_len = max(max_len, end - start + 1)

    return max_len

print(longest_unique_substring("abcabcbb"))  # 3 ("abc")
print(longest_unique_substring("pwwkew"))    # 3 ("wke")

가변 크기 윈도우: 합이 목표값 이상인 최소 부분 배열

def min_subarray_sum(arr, target):
    """Return the length of the smallest subarray with sum >= target, or 0."""
    min_len = float('inf')
    window_sum = 0
    start = 0

    for end in range(len(arr)):
        window_sum += arr[end]             # Expand window

        while window_sum >= target:        # Shrink while condition holds
            min_len = min(min_len, end - start + 1)
            window_sum -= arr[start]
            start += 1

    return min_len if min_len != float('inf') else 0

print(min_subarray_sum([2, 3, 1, 2, 4, 3], 7))  # 2 ([4, 3])
고정 크기 슬라이딩 윈도우(k=3)가 배열을 가로질러 이동합니다. 각 위치에서 새 원소를 더하고 윈도우를 떠난 원소를 빼어 O(1)로 합이 업데이트됩니다. 최대 합 윈도우가 강조 표시됩니다.

탐욕 알고리즘 (Greedy)

탐욕(greedy) 알고리즘은 전역 최적을 찾기를 바라며 각 단계에서 지역적으로 최적인 선택을 합니다. 탐욕은 문제에 탐욕적 선택 속성(greedy choice property)이 있을 때 작동합니다 — 지역적으로 최적인 선택이 전역적으로 최적인 해법으로 이어집니다.

탐욕이 작동하는 경우: 구간 스케줄링

문제: 구간(시작, 끝) 집합이 주어졌을 때, 겹치지 않는 최대 구간 수를 찾으십시오.

전략: 가장 일찍 끝나는 구간을 항상 선택합니다. 이렇게 하면 미래의 구간을 위한 공간이 가장 많이 남습니다.

def max_non_overlapping(intervals):
    """Return max number of non-overlapping intervals.
    Each interval is (start, end).
    """
    # Sort by end time
    intervals.sort(key=lambda x: x[1])

    count = 0
    last_end = float('-inf')

    for start, end in intervals:
        if start >= last_end:      # No overlap with previous selection
            count += 1
            last_end = end         # Update the end boundary

    return count

meetings = [(1, 3), (2, 5), (3, 6), (5, 7), (8, 10), (6, 9)]
print(max_non_overlapping(meetings))  # 3: (1,3), (5,7), (8,10)

탐욕이 실패하는 경우: 특이한 액면가의 동전 거스름돈

탐욕은 미국 동전 액면가(25, 10, 5, 1)에 대해서는 작동합니다: 항상 맞는 가장 큰 동전을 선택합니다. 하지만 액면가 [1, 3, 4]와 금액 6을 생각해 보십시오:

  • 탐욕: 4 + 1 + 1 = 동전 3개
  • 최적: 3 + 3 = 동전 2개

탐욕 접근법은 가장 큰 동전을 먼저 선택하지만, 그것이 더 나은 해법을 막습니다. 25장에서 일반 동전 거스름돈 문제에 DP가 필요했던 이유입니다.

주의: 탐욕이 작동하는 것처럼 보이지만 그렇지 않을 때

탐욕 알고리즘의 위험은 많은 입력에서 거의 작동한다는 것입니다. 여러 예시로 테스트하여 올바른 결과를 얻다가, 프로덕션에서 반례를 만날 수 있습니다. 탐욕이 문제에 올바르다는 것을 항상 증명하거나, 확실하지 않으면 DP를 사용하십시오. AI 생성 해법은 DP가 필요할 때 탐욕을 사용하는 경우가 많습니다 — 이를 주의하십시오.

분할 정복 (Divide and Conquer)

분할 정복(divide and conquer)은 문제를 독립적인 부분 문제로 분할하고, 각각을 재귀적으로 풀고, 결과를 합칩니다. 이미 병합 정렬(24장)을 알고 있습니다. 이 패턴은 정렬 이상에도 적용됩니다:

  • 가장 가까운 점 쌍—점들을 x좌표로 분할하고, 각 절반에서 가장 가까운 쌍을 찾은 후, 분할선을 가로질러 확인합니다.
  • 역순 쌍 세기—순서가 잘못된 쌍이 몇 개인지 세는 수정된 병합 정렬.
  • 카라추바 곱셈(Karatsuba multiplication)—큰 숫자를 절반으로 나누어 곱하여, O(n²)에서 O(n¹·⁵⁸)로 줄입니다.

DP와의 핵심 차이: 분할 정복의 부분 문제는 독립적입니다(중복 없음). DP의 부분 문제는 중복됩니다.

백트래킹 (Backtracking)

백트래킹(backtracking)은 가능한 모든 선택을 탐색하지만, 유효한 해법으로 이어질 수 없는 가지를 가지치기(pruning)합니다. 막다른 길에서 시간을 낭비하지 않고 모든 것을 체계적으로 시도하는 방법입니다.

패턴: 선택하고, 재귀하고, 선택을 취소하고(백트래킹), 다음 선택지를 시도합니다.

예시: 모든 부분 집합 생성

def subsets(nums):
    """Generate all subsets of nums."""
    result = []

    def backtrack(start, current):
        result.append(current[:])          # Add a copy of the current subset

        for i in range(start, len(nums)):
            current.append(nums[i])        # Choose
            backtrack(i + 1, current)      # Explore
            current.pop()                  # Un-choose (backtrack)

    backtrack(0, [])
    return result

print(subsets([1, 2, 3]))
# [[], [1], [1,2], [1,2,3], [1,3], [2], [2,3], [3]]

예시: N-퀸 (n×n 보드에 n개의 퀸을 배치할 수 있는지 결정)

def solve_n_queens(n):
    """Return the number of ways to place n queens on an n x n board
    so that no two queens attack each other."""
    solutions = 0
    cols = set()          # Columns with queens
    diag1 = set()         # "row - col" diagonals with queens
    diag2 = set()         # "row + col" diagonals with queens

    def backtrack(row):
        nonlocal solutions
        if row == n:               # All queens placed successfully
            solutions += 1
            return

        for col in range(n):
            if col in cols or (row - col) in diag1 or (row + col) in diag2:
                continue           # Prune: this position is attacked

            # Place queen
            cols.add(col)
            diag1.add(row - col)
            diag2.add(row + col)

            backtrack(row + 1)     # Try next row

            # Remove queen (backtrack)
            cols.remove(col)
            diag1.remove(row - col)
            diag2.remove(row + col)

    backtrack(0)
    return solutions

print(solve_n_queens(8))  # 92 solutions

def max_completions_in_window(completion_days, window_size=7):
    """Return the maximum number of tasks completed in any window_size-day period.
    completion_days is a sorted list of day numbers when tasks were completed.
    """
    if not completion_days:
        return 0

    max_count = 0
    start = 0

    for end in range(len(completion_days)):
        # Shrink window: remove tasks outside the window
        while completion_days[end] - completion_days[start] >= window_size:
            start += 1
        # All tasks from start to end are within the window
        max_count = max(max_count, end - start + 1)

    return max_count

너비 우선 탐색 (BFS, Breadth-First Search)

너비 우선 탐색(breadth-first search, BFS)은 그래프를 레벨별로 탐색하여, 다음 레벨로 이동하기 전에 현재 노드의 모든 이웃을 방문합니다. 큐(queue)(FIFO) — 20장의 자료 구조 — 를 사용합니다. BFS는 시작점으로부터 거리 순서대로 노드를 발견하기 때문에 비가중 그래프에서 최단 경로를 찾습니다.

from collections import deque

def bfs(graph, start):
    """Perform BFS from start node. Return dict of {node: distance}.
    graph: dict of {node: [neighbors]}
    """
    visited = {start: 0}              # node -> distance from start
    queue = deque([start])

    while queue:
        node = queue.popleft()         # Dequeue the front node
        for neighbor in graph[node]:
            if neighbor not in visited:
                visited[neighbor] = visited[node] + 1
                queue.append(neighbor) # Enqueue unvisited neighbor

    return visited

# Example graph:
#   A -- B -- D
#   |         |
#   C -- E -- F
graph = {
    'A': ['B', 'C'],
    'B': ['A', 'D'],
    'C': ['A', 'E'],
    'D': ['B', 'F'],
    'E': ['C', 'F'],
    'F': ['D', 'E'],
}
distances = bfs(graph, 'A')
print(distances)
# {'A': 0, 'B': 1, 'C': 1, 'D': 2, 'E': 2, 'F': 3}

from collections import deque

def bfs_path(graph, start, end):
    """Return the shortest path from start to end, or None."""
    if start == end:
        return [start]

    visited = {start}
    queue = deque([(start, [start])])  # (node, path_so_far)

    while queue:
        node, path = queue.popleft()
        for neighbor in graph[node]:
            if neighbor == end:
                return path + [neighbor]
            if neighbor not in visited:
                visited.add(neighbor)
                queue.append((neighbor, path + [neighbor]))

    return None  # No path exists

print(bfs_path(graph, 'A', 'F'))  # ['A', 'B', 'D', 'F'] or ['A', 'C', 'E', 'F']

깊이 우선 탐색 (DFS, Depth-First Search)

깊이 우선 탐색(depth-first search, DFS)은 백트래킹하기 전에 각 분기를 가능한 한 깊이 탐색합니다. 스택(stack)(LIFO) — 호출 스택(재귀를 통해) 또는 명시적 스택 자료 구조 — 을 사용합니다.

재귀적 DFS

def dfs_recursive(graph, node, visited=None):
    """Perform DFS from node. Return list of visited nodes in order."""
    if visited is None:
        visited = []
    visited.append(node)

    for neighbor in graph[node]:
        if neighbor not in visited:
            dfs_recursive(graph, neighbor, visited)

    return visited

print(dfs_recursive(graph, 'A'))  # ['A', 'B', 'D', 'F', 'E', 'C']

반복적 DFS

def dfs_iterative(graph, start):
    """Perform DFS from start using an explicit stack."""
    visited = []
    stack = [start]

    while stack:
        node = stack.pop()
        if node not in visited:
            visited.append(node)
            # Add neighbors in reverse to visit them in order
            for neighbor in reversed(graph[node]):
                if neighbor not in visited:
                    stack.append(neighbor)

    return visited

print(dfs_iterative(graph, 'A'))  # ['A', 'B', 'D', 'F', 'E', 'C']

DFS 응용:

  • 사이클 감지(cycle detection)—방향 그래프에서 DFS 중, 현재 재귀 스택에 있는 노드(단순히 방문된 것이 아닌)를 만나면 사이클이 있습니다.
  • 위상 정렬(topological sort)—아래에서 다룹니다.
  • 연결된 컴포넌트(connected components)—방문하지 않은 각 노드에서 DFS를 실행합니다; 각 DFS는 하나의 컴포넌트를 발견합니다.
  • 경로 탐색(path finding)—DFS는 하나의 경로를 찾지만, 반드시 최단 경로는 아닙니다.
BFS(왼쪽)는 노드를 레벨별로 방문합니다 — 거리 2 노드들 전에 거리 1 노드들을 먼저. DFS(오른쪽)는 백트래킹하기 전에 한 분기를 깊이 파고듭니다. 숫자는 방문 순서를 나타냅니다.

다익스트라 알고리즘 (Dijkstra's Algorithm)

다익스트라 알고리즘(Dijkstra's algorithm)은 비음수 가중치의 가중 그래프에서 최단 경로를 찾습니다. 우선순위 큐(priority queue)(22장의 최소 힙)를 사용하여 항상 가장 가까운 미방문 노드를 먼저 처리합니다.

알고리즘을 단계별로 살펴보면:

  1. 시작 노드까지의 거리를 0으로, 다른 모든 거리는 무한대로 설정합니다.
  2. 시작 노드를 거리 0으로 우선순위 큐에 추가합니다.
  3. 우선순위 큐가 비어있지 않은 동안:
    1. 가장 작은 거리의 노드를 추출합니다.
    2. 각 이웃에 대해, 현재 노드를 통한 거리를 계산합니다.
    3. 이 거리가 이웃의 현재 거리보다 짧으면, 업데이트하고 이웃을 큐에 추가합니다.
import heapq

def dijkstra(graph, start):
    """Find shortest distances from start to all nodes.
    graph: dict of {node: [(neighbor, weight), ...]}
    Returns: dict of {node: shortest_distance}
    """
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    pq = [(0, start)]             # (distance, node)
    visited = set()

    while pq:
        dist, node = heapq.heappop(pq)

        if node in visited:        # Already processed with a shorter distance
            continue
        visited.add(node)

        for neighbor, weight in graph[node]:
            new_dist = dist + weight
            if new_dist < distances[neighbor]:
                distances[neighbor] = new_dist
                heapq.heappush(pq, (new_dist, neighbor))

    return distances

# Weighted graph:
#   A --4-- B --2-- D
#   |       |       |
#   1       3       1
#   |       |       |
#   C --5-- E --1-- F
weighted = {
    'A': [('B', 4), ('C', 1)],
    'B': [('A', 4), ('D', 2), ('E', 3)],
    'C': [('A', 1), ('E', 5)],
    'D': [('B', 2), ('F', 1)],
    'E': [('B', 3), ('C', 5), ('F', 1)],
    'F': [('D', 1), ('E', 1)],
}
print(dijkstra(weighted, 'A'))
# {'A': 0, 'B': 4, 'C': 1, 'D': 6, 'E': 6, 'F': 7}

주의: 다익스트라는 비음수 가중치가 필요합니다

그래프에 음수 간선 가중치가 있으면, 다익스트라는 잘못된 결과를 생성합니다. 대신 벨만-포드(Bellman-Ford) 알고리즘을 사용하십시오(O(V × E)). 음수 가중치는 금융 차익거래 감지와 일부 게임 메커니즘에 등장하지만, 대부분의 실제 그래프(도로 거리, 네트워크 지연)는 비음수 가중치를 가집니다.

최단 경로 재구성

import heapq

def dijkstra_path(graph, start, end):
    """Find the shortest path and its distance from start to end."""
    distances = {node: float('inf') for node in graph}
    distances[start] = 0
    previous = {node: None for node in graph}  # Track the path
    pq = [(0, start)]
    visited = set()

    while pq:
        dist, node = heapq.heappop(pq)
        if node in visited:
            continue
        visited.add(node)

        if node == end:
            break

        for neighbor, weight in graph[node]:
            new_dist = dist + weight
            if new_dist < distances[neighbor]:
                distances[neighbor] = new_dist
                previous[neighbor] = node
                heapq.heappush(pq, (new_dist, neighbor))

    # Reconstruct path
    path = []
    current = end
    while current is not None:
        path.append(current)
        current = previous[current]
    path.reverse()

    return path, distances[end]

path, dist = dijkstra_path(weighted, 'A', 'F')
print(f"Path: {path}, Distance: {dist}")
# Path: ['A', 'B', 'D', 'F'], Distance: 7

위상 정렬 (Topological Sort)

위상 정렬(topological sort)은 모든 간선 u → v에 대해 u가 v 앞에 나타나도록 방향 비순환 그래프(DAG, Directed Acyclic Graph)의 노드들을 선형으로 정렬합니다. 이것이 의존성을 해결하는 알고리즘입니다: 빌드 시스템, 패키지 관리자, 과목 선수 조건, 작업 스케줄링.

칸 알고리즘 (Kahn's Algorithm, BFS 기반)

칸 알고리즘은 진입 차수(in-degree) 추적을 사용합니다: 들어오는 간선이 없는 노드(진입 차수 0)에서 시작하고, 처리하고, 이웃의 진입 차수를 줄이고, 반복합니다.

from collections import deque

def topological_sort_kahn(graph):
    """Return a topological ordering of the DAG, or None if a cycle exists.
    graph: dict of {node: [dependents]} (edges point from dependency to dependent)
    """
    # Step 1: Compute in-degrees
    in_degree = {node: 0 for node in graph}
    for node in graph:
        for neighbor in graph[node]:
            in_degree[neighbor] = in_degree.get(neighbor, 0) + 1

    # Step 2: Start with all nodes that have in-degree 0
    queue = deque([node for node in graph if in_degree[node] == 0])
    result = []

    # Step 3: Process nodes
    while queue:
        node = queue.popleft()
        result.append(node)

        for neighbor in graph[node]:
            in_degree[neighbor] -= 1       # One dependency satisfied
            if in_degree[neighbor] == 0:   # All dependencies satisfied
                queue.append(neighbor)

    # Step 4: Check for cycles
    if len(result) != len(graph):
        return None  # Cycle detected! Not all nodes were processed.

    return result

# TaskForge dependency DAG:
#   "setup" -> "backend" -> "deploy"
#   "setup" -> "frontend" -> "deploy"
#   "backend" -> "tests"
deps = {
    "setup":    ["backend", "frontend"],
    "backend":  ["tests", "deploy"],
    "frontend": ["deploy"],
    "tests":    [],
    "deploy":   [],

    in_progress = set()  # For cycle detection

    def dfs(node):
        if node in in_progress:
            return False           # Cycle detected
        if node in visited:
            return True

        in_progress.add(node)
        for neighbor in graph[node]:
            if not dfs(neighbor):
                return False
        in_progress.remove(node)
        visited.add(node)
        stack.append(node)         # Post-order: add after all descendants
        return True

    for node in graph:
        if node not in visited:
            if not dfs(node):
                return None        # Cycle

    stack.reverse()                # Reverse post-order = topological order
    return stack

print(topological_sort_dfs(deps))
# ['setup', 'frontend', 'backend', 'tests', 'deploy']

복잡도: 두 알고리즘 모두 O(V + E)로 실행됩니다.

유니온-파인드 (Union-Find, 서로소 집합)

유니온-파인드(Union-Find)는 연결된 컴포넌트를 효율적으로 추적합니다. 두 가지 연산을 지원합니다:

  • find(x)—x의 컴포넌트의 루트 대표를 반환합니다.
  • union(x, y)—x와 y를 포함하는 컴포넌트를 병합합니다.

경로 압축(path compression)(find 중 트리를 평탄화)과 랭크에 의한 유니온(union by rank)(더 작은 트리를 더 큰 트리 아래에 붙임)을 사용하면, 두 연산 모두 거의 상수 시간으로 실행됩니다: 분할 상환 O(α(n)), 여기서 α는 역 아커만 함수(inverse Ackermann function)입니다 — 실질적으로 모든 목적에서 O(1).

class UnionFind:
    """Disjoint Set with path compression and union by rank."""

    def __init__(self, n):
        self.parent = list(range(n))   # Each node is its own parent initially
        self.rank = [0] * n            # Rank (approximate tree height)
        self.components = n            # Number of distinct components

    def find(self, x):
        """Return the root of x's component (with path compression)."""
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])  # Flatten the tree
        return self.parent[x]

    def union(self, x, y):
        """Merge the components of x and y. Return True if they were separate."""
        rx, ry = self.find(x), self.find(y)
        if rx == ry:
            return False               # Already in the same component

        # Attach smaller tree under larger tree
        if self.rank[rx] < self.rank[ry]:
            self.parent[rx] = ry
        elif self.rank[rx] > self.rank[ry]:
            self.parent[ry] = rx
        else:
            self.parent[ry] = rx
            self.rank[rx] += 1

        self.components -= 1
        return True

    def connected(self, x, y):
        """Check if x and y are in the same component."""
        return self.find(x) == self.find(y)

# Example: network of 6 servers
uf = UnionFind(6)
uf.union(0, 1)   # Connect server 0 and 1
uf.union(2, 3)   # Connect server 2 and 3
uf.union(1, 3)   # Connect the two clusters

print(uf.connected(0, 3))  # True — all four are connected
print(uf.connected(0, 4))  # False — server 4 is isolated
print(uf.components)        # 3 (cluster {0,1,2,3}, isolated 4, isolated 5)

사용 사례: 간선 추가가 사이클을 만드는지 감지(크루스칼의 MST 알고리즘), 네트워크 연결성 확인, 관련 항목 그룹화(예: 중복 사진 감지).

문제 해결 프레임워크

그래프 문제를 만났을 때, 이 체계적인 접근법을 사용하십시오:

  1. 문제 이해—노드는 무엇입니까? 간선은 무엇입니까? 방향이 있습니까? 가중치가 있습니까? 무엇을 찾고 있습니까?
  2. 형태 파악:
    • 비가중 그래프에서 최단 경로 → BFS
    • 가중 그래프에서 최단 경로(비음수) → 다익스트라
    • 의존성 순서 지정 → 위상 정렬
    • 컴포넌트 연결성 → 유니온-파인드 또는 BFS/DFS
    • 사이클 감지(방향) → 진행 중 추적과 함께 DFS
    • 사이클 감지(무방향) → 유니온-파인드
    • 모든 경로 / 모든 가능성 탐색 → DFS + 백트래킹
  3. 구현 및 검증—엣지 케이스로 테스트합니다: 빈 그래프, 단일 노드, 연결되지 않은 컴포넌트, 사이클.

TaskForge: 위상 정렬로 의존성 해결

의존성이 있는 TaskForge 작업들이 주어졌을 때, 위상 정렬을 구현하여 유효한 실행 순서를 찾으십시오. 유효한 순서가 없으면(의존성 그래프의 사이클), 오류를 보고합니다.

from collections import deque

def resolve_task_order(tasks):
    """Given a dict of {task: [dependencies]}, return a valid execution order.
    Raises ValueError if there is a circular dependency.
    """
    # Build the forward graph and compute in-degrees
    graph = {t: [] for t in tasks}
    in_degree = {t: 0 for t in tasks}

    for task, deps in tasks.items():
        for dep in deps:
            graph[dep].append(task)    # dep must come before task
            in_degree[task] += 1

    # Kahn's algorithm
    queue = deque([t for t in tasks if in_degree[t] == 0])
    order = []

    while queue:
        task = queue.popleft()
        order.append(task)
        for dependent in graph[task]:
            in_degree[dependent] -= 1
            if in_degree[dependent] == 0:
                queue.append(dependent)

    if len(order) != len(tasks):
        raise ValueError("Circular dependency detected! Cannot determine order.")

    return order

# TaskForge sprint tasks with dependencies
sprint = {
    "Set up database":    [],
    "Build API":          ["Set up database"],
    "Build frontend":     ["Build API"],
    "Write tests":        ["Build API"],
    "Deploy":             ["Build frontend", "Write tests"],
}

print(resolve_task_order(sprint))
# ['Set up database', 'Build API', 'Build frontend', 'Write tests', 'Deploy']

# Detect circular dependency
try:
    circular = {
        "A": ["B"],
        "B": ["C"],
        "C": ["A"],  # Cycle!
    }
    resolve_task_order(circular)
except ValueError as e:
    print(e)  # "Circular dependency detected! Cannot determine order."
