DFS Engine Build Guide — April 2026

A modern DFS prop betting automation engine from scratch

Build instructions for an LLM creating a +EV identification, slip generation, and automated submission system targeting PrizePicks and Underdog Fantasy.

15 build stepsTypeScript strictCLI tool
Overview

What you are building

An automated system that pulls player prop lines from DFS platforms, compares them against sharp sportsbook odds, identifies +EV edges, generates optimal slip combinations, and submits them via headless browser automation.

Data Pipeline

Pull and compare lines

Fetch player props from PrizePicks and Underdog Fantasy, pull sharp odds from sportsbook aggregators, and find divergences.

Analysis Engine

Probability models and scoring

Normal CDF, Monte Carlo simulation, multi-factor scoring with stochastic rolls, and combinatorial optimization with player and stat caps.

Automation

Submit and track

Headless browser submission via Steel.dev with anti-detection, plus Supabase persistence for backtesting and P&L monitoring.

For LLMs

These instructions are designed for an LLM building this project from scratch. The reference codebase is the original “better/golflifters” repo.

Step 01

Tech stack

A modern Node.js CLI tool with TypeScript strict mode, no web framework.

Stack 
Runtime:         Node.js 22 LTS (ES modules)
Language:        TypeScript 5.x (strict mode)
Browser:         Playwright + Steel.dev cloud sessions
HTTP Client:     undici (Node built-in)
Database:        Supabase (PostgreSQL)
Cache:           Redis 7+ with ioredis
Odds Data:       The Odds API + OddsPapi
DFS Lines:       PrizePicks API + Underdog scraper
Stats Data:      nba_api (Python), DataGolf API, ESPN API
Math:            @stdlib/stats, simple-statistics
Formatting:      Biome (replaces Prettier + ESLint)
Testing:         Vitest
CI:              GitHub Actions
Package Manager: pnpm
Step 02

Project structure

Create this exact directory structure. Every platform normalizer produces Line objects. Every engine function operates on Line[] and Slip[].
