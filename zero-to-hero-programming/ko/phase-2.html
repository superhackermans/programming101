함수(Functions)

함수(function)는 주방의 레시피 카드와 같습니다. 이름, 재료 목록(매개변수), 단계별 지시 사항이 있습니다. 레시피를 한 번 작성해두면 그 요리가 필요할 때마다 사용할 수 있습니다. 코드로 말하면: 한 번 정의하고, 필요할 때마다 호출합니다. 함수 하나, 역할 하나.

def greet(name):
    return f"Hello, {name}!"

print(greet("World"))  # Hello, World!

def는 함수를 정의합니다. name은 매개변수(parameter)—입력값입니다. return은 값을 돌려줍니다. 함수를 정의하는 것만으로는 실행되지 않습니다; 호출(call)해야 실행됩니다.

조건문(Conditionals)

if/elif/else는 프로그램이 경로를 선택할 수 있게 해줍니다. 프로그램은 첫 번째 참인 분기를 실행하고 나머지는 건너뜁니다.

프로그램은 하나의 경로만 선택합니다. 한 분기에 진입하면 나머지는 모두 건너뜁니다.

반복문(Loops)

for 반복문은 컬렉션의 각 항목을 처리합니다. while 반복문은 조건이 바뀔 때까지 반복합니다.

반복문은 한 번에 하나의 항목을 처리합니다. 누산기(accumulator)가 누적된 결과를 기억합니다.

스코프(Scope)

함수 내부에서 정의된 변수는 외부에서 보이지 않습니다. 이를 스코프(scope)라고 합니다. AI가 생성한 코드에서 자주 발생하는 버그의 원인으로, AI가 다른 함수 내부에만 존재하는 변수를 참조하는 경우가 있습니다.

구체적인 예시입니다. 이 함수는 카운터를 업데이트하는 것처럼 보이지만, 실제로는 그렇지 않습니다:

# 버그: 외부 변수를 수정하지 않습니다
count = 0

def increment():
    count = count + 1  # UnboundLocalError!

increment()
print(count)  # 여기에 도달하지 못합니다

Python은 함수 안의 count = ...를 보고 count를 지역(local) 변수로 취급합니다. 그런데 오른쪽에서 아직 로컬로 할당되지 않은 count를 읽으려 하기 때문에 오류가 발생합니다. 해결책: 값을 인수로 전달하고 결과를 반환합니다.

# 해결: 인수로 전달하고, 결과를 반환합니다
count = 0

def increment(current):
    return current + 1

count = increment(count)
print(count)  # 1
주의

AI가 생성한 코드는 전역 상태(global state)를 조작하는 경우가 있습니다. 스코프 문제를 인식하면 이런 버그를 잡는 데 도움이 됩니다. 함수의 매개변수도 아니고 함수 내부에서 정의되지도 않은 변수를 함수가 읽거나 쓰고 있다면, 그건 경고 신호입니다.

문자열 메서드(String Methods)

문자열(string)은 어디에나 있습니다—사용자 입력, 파일 경로, API 응답, 로그 메시지. Python은 문자열을 자르고, 정리하고, 재조합하는 내장 메서드를 제공합니다. 텍스트 데이터를 처리하거나 AI 출력을 파싱할 때 이것들을 끊임없이 사용하게 됩니다.

f-문자열(f-strings)

f-문자열을 사용하면 표현식을 문자열 안에 직접 삽입할 수 있습니다. 문자열 앞에 f를 붙이고 중괄호 안에 변수나 표현식을 넣으면 됩니다:

name = "Alice"

print(f"{name} has {tasks} tasks")  # Alice has 5 tasks
print(f"Double: {tasks * 2}")       # Double: 10

핵심 확인

함수 내부에서 정의된 변수를 함수 바깥에서 사용하려 하면 어떻게 됩니까?

값으로서의 함수: 고차 함수(Higher-Order Functions)와 클로저(Closures)

Python에서 함수는 일급 값(first-class values)입니다 — 변수에 할당하거나, 인수로 전달하거나, 다른 함수에서 반환할 수 있습니다. AI는 이러한 패턴을 끊임없이 생성합니다.

내장 고차 함수(Built-in Higher-Order Functions)

key 함수와 함께 사용하는 map(), filter(), sorted()가 가장 흔합니다. 각각 함수를 인수로 받습니다:

# map: 모든 요소에 함수 적용
names = ["alice", "bob", "charlie"]
upper_names = list(map(str.upper, names))  # ["ALICE", "BOB", "CHARLIE"]

# filter: 함수가 True를 반환하는 요소만 유지
numbers = [1, 2, 3, 4, 5, 6]
evens = list(filter(lambda x: x % 2 == 0, numbers))  # [2, 4, 6]

# sorted with key: 사용자 정의 기준으로 정렬
tasks = [{"title": "B task", "priority": 2}, {"title": "A task", "priority": 1}]
by_priority = sorted(tasks, key=lambda t: t["priority"])
참고

AI는 map()과 filter()를 자주 생성합니다. Python에서는 리스트 컴프리헨션(list comprehension)이 보통 더 읽기 쉽습니다: list(map(str.upper, names)) vs [x.upper() for x in names]. 둘 다 알면 AI 코드를 읽고 어느 쪽이 상황에 더 명확한지 판단할 수 있습니다.

람다 표현식(Lambda Expressions)

lambda는 익명 함수(anonymous function)입니다 — 짧고 일회성 인수로 유용합니다:

# Lambda: 일회성 익명 함수
square = lambda x: x ** 2  # def square(x): return x ** 2 와 동일

# 고차 함수의 인수로 가장 유용합니다
sorted_tasks = sorted(tasks, key=lambda t: t["due_date"])

경험 법칙: lambda가 표현식 하나를 초과하면 이름 있는 함수를 사용하세요. 간결함보다 가독성이 중요합니다.

실제로 PEP 8은 이름 있는 함수에 def를 권장합니다. lambda는 인라인 인수로 빛납니다: sorted(names, key=lambda n: len(n)).

클로저(Closures)

클로저(closure)는 정의된 스코프의 변수를 기억하는 함수입니다:

def make_multiplier(n):
    def multiplier(x):
        return x * n  # 'n'은 외부 스코프에서 기억됩니다
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)
print(double(5))  # 10
print(triple(5))  # 15

AI는 콜백(callback), 이벤트 핸들러(event handler), 팩토리 패턴(factory pattern)에서 클로저를 생성합니다. 이를 이해하면 이런 패턴이 나타났을 때 읽고 디버그할 수 있습니다.

모든 요소에 함수를 적용하는 apply_to_all(func, lst)(map처럼)와, func가 True를 반환하는 요소만 유지하는 keep_if(func, lst)(filter처럼)를 구현하세요.

apply_to_all: 새 리스트를 만들고, lst를 순회하며 각 항목에 func(item)을 추가하세요.

keep_if: 새 리스트를 만들고, lst를 순회하며 func(item)이 True일 때만 항목을 추가하세요.

apply_to_all: return [func(x) for x in lst]. keep_if: return [x for x in lst if func(x)].

핵심 개념

모든 API 응답, 모든 설정 파일, 모든 AI 도구 출력은 구조화된 데이터입니다—보통 JSON 형식이며, 이는 중첩된 리스트와 딕셔너리입니다. 중첩 데이터를 탐색하지 못하면 API를 사용하거나, AI 출력을 읽거나, 데이터 문제를 디버그할 수 없습니다.

함수는 데이터를 처리합니다. 그런데 데이터는 어디에 존재할까요? 자료 구조(data structures)—정보를 구성하는 컨테이너에 있습니다. Python과 AI 도구에서 두 가지 구조가 지배적입니다: 리스트(list)(순서가 있는 시퀀스)와 딕셔너리(dictionary)(키-값 쌍).

리스트(Lists)

딕셔너리(Dictionaries)

딕셔너리(dictionary)는 키를 값에 매핑합니다. API와 AI 도구에서 가장 중요한 구조입니다. 보편적 데이터 형식인 JSON은 본질적으로 중첩된 딕셔너리이기 때문입니다.

리스트 컴프리헨션(List Comprehensions)

리스트 컴프리헨션(list comprehension)은 한 줄로 새 리스트를 만듭니다. 리스트에 추가하는 for 반복문의 간결한 대안입니다. AI가 생성하는 Python은 컴프리헨션을 많이 사용하므로 유창하게 읽을 수 있어야 합니다.

기본 문법

[expression for item in iterable]

이렇게 읽힙니다: "이터러블(iterable)의 각 item에 대해 expression을 평가하고 결과를 리스트로 모읍니다."

조건 포함

[expression for item in iterable if condition]

필터를 추가합니다: condition이 참인 항목만 포함됩니다.

전후 비교

동일한 로직을 반복문과 컴프리헨션으로 작성한 것입니다:

# 반복문 버전
squares = []
for n in range(5):
    squares.append(n ** 2)

# 컴프리헨션 버전
squares = [n ** 2 for n in range(5)]
# 둘 다 결과: [0, 1, 4, 9, 16]
# 필터가 있는 반복문
evens = []
for n in range(10):
    if n % 2 == 0:
        evens.append(n)

# 필터가 있는 컴프리헨션
evens = [n for n in range(10) if n % 2 == 0]
# 둘 다 결과: [0, 2, 4, 6, 8]

챕터 07 클래스(Classes)와 객체 지향 프로그래밍(OOP) 기초

핵심 개념

AI가 생성한 Python 코드는 클래스를 자주 사용합니다. Claude가 self.tasks가 있는 class Task:를 작성하면, 무엇을 읽고 있는지 이해해야 합니다.

지금까지 딕셔너리에 데이터를 저장하고 함수로 처리했습니다. 이 방식은 효과적이지만—코드베이스 전반에 걸쳐 데이터와 이를 처리하는 함수가 점점 떨어지기 시작할 때까지만 그렇습니다. 클래스(class)는 데이터와 동작을 객체(object)라는 단일 단위로 묶어 이 문제를 해결합니다.

클래스란 무엇인가?

클래스(class)는 객체를 만드는 청사진입니다. 쿠키 커터와 같습니다: 클래스가 모양을 정의하고, 찍어낸 각 쿠키가 인스턴스(instance)—그 청사진으로 만들어진 개별 객체입니다. 쿠키 커터 자체는 쿠키가 아닙니다; 쿠키를 만드는 템플릿입니다. 모든 쿠키는 같은 모양이지만, 각각 다른 장식(다른 데이터)을 가질 수 있습니다.

프로그래밍 용어로: 클래스는 모든 인스턴스가 가질 속성(attribute, 데이터)과 메서드(method, 동작)를 정의합니다. 각 인스턴스는 자체 데이터 복사본을 갖지만, 모두 같은 메서드를 공유합니다.

클래스 정의하기

가장 간단한 유용한 클래스입니다:

class Task:
    def __init__(self, title):
        self.title = title
        self.done = False

하나씩 분해해 봅시다:

  • class Task: — Task라는 새 클래스를 선언합니다. 클래스 이름은 관례상 CamelCase를 사용합니다(함수의 snake_case와 다릅니다).
  • __init__ — 초기화자(initializer)(생성자(constructor)라고도 합니다). Python은 새 인스턴스를 만들 때 이것을 자동으로 호출합니다. 이중 밑줄("dunder"라고 부름)은 특수 메서드임을 나타냅니다.
  • self — 생성 중인 특정 인스턴스에 대한 참조입니다. self.title = title을 쓰면 "이 특정 객체의 title은 전달된 값이다"라고 말하는 것입니다. 클래스의 모든 메서드는 첫 번째 매개변수로 self를 받습니다.
  • self.title과 self.done — 이것들은 인스턴스 속성(instance attributes)입니다. 각 Task 객체는 자체 title과 done을 갖습니다.

메서드 vs 함수

메서드(method)는 클래스에 속하는 함수입니다. 문법적 차이는 하나: 메서드는 첫 번째 매개변수로 self를 받아 인스턴스의 데이터에 접근합니다. 일반 함수는 독립적으로 존재하고; 메서드는 객체에 붙어 그 객체의 데이터를 조작합니다.

# 일반 함수 — 독립적으로 존재
def mark_done(task_dict):
    task_dict["done"] = True

# 메서드 — 클래스에 속하며 self에서 동작
class Task:
    def __init__(self, title):
        self.title = title
        self.done = False

    def mark_done(self):
        self.done = True

task.mark_done()을 호출하면 Python이 자동으로 인스턴스를 self로 전달합니다. task.mark_done(task)라고 쓸 필요가 없습니다—Python이 처리합니다.

인스턴스 생성하기

클래스를 함수처럼 호출해서 인스턴스를 만듭니다:

task = Task("Buy groceries")
print(task.title)  # Buy groceries
print(task.done)   # False

"Buy groceries"를 전달하지만 self는 전달하지 않습니다. Python이 self를 자동으로 채웁니다. Task(...)를 호출할 때마다 완전히 새로운, 독립적인 객체가 만들어집니다.

인스턴스 속성

모든 인스턴스는 자체 데이터를 갖습니다. 하나의 객체를 변경해도 다른 것에 영향을 미치지 않습니다:

task1 = Task("Buy groceries")
task2 = Task("Write report")

task1.mark_done()
print(task1.done)  # True
print(task2.done)  # False — 완전히 독립적

점 표기법(dot notation)으로 속성에 접근합니다: task.title, task.done. 이것은 문자열("hello".upper())과 리스트(my_list.append(x))에서 사용해온 것과 같은 문법입니다—문자열과 리스트도 객체이기 때문입니다.

완성된 예시

초기화자, 동작 메서드, 문자열 표현이 있는 Task 클래스입니다:

class Task:
    def __init__(self, title):
        self.title = title
        self.done = False

    def mark_done(self):
        self.done = True

    def __str__(self):
        status = "done" if self.done else "pending"
        return f"[{status}] {self.title}"

# 사용 예
task = Task("Buy groceries")
print(task)          # [pending] Buy groceries
task.mark_done()
print(task)          # [done] Buy groceries

__str__은 또 다른 dunder 메서드입니다. Python은 객체를 print()하거나 문자열로 변환할 때 이것을 자동으로 호출합니다. 없으면 Task를 출력했을 때 <__main__.Task object at 0x...> 같은 도움이 안 되는 내용이 표시됩니다.

클래스(class)는 세 칸으로 이루어진 상자입니다: 맨 위에 이름, 가운데에 데이터(속성), 아래에 동작(메서드). 이것이 UML 클래스 다이어그램 패턴입니다.

상속(Inheritance)의 기초

기존 클래스와 거의 같지만 몇 가지만 추가해야 할 때가 있습니다. 상속(inheritance)을 사용하면 기존 클래스를 기반으로 새 클래스를 만들 수 있습니다. 새 클래스(자식 클래스 또는 서브클래스(subclass))는 기존 클래스(부모 클래스 또는 슈퍼클래스(superclass))의 모든 속성과 메서드를 물려받으며, 추가하거나 재정의할 수 있습니다.

다음은 Task에 우선순위 필드를 추가한 PriorityTask 예시입니다:

class PriorityTask(Task):
    def __init__(self, title, priority="medium"):
        super().__init__(title)
        self.priority = priority

    def __str__(self):
        status = "done" if self.done else "pending"
        return f"[{status}] ({self.priority}) {self.title}"

class PriorityTask(Task)에서 괄호는 "PriorityTask가 Task를 상속한다"는 뜻입니다. super().__init__(title) 호출은 부모의 __init__을 실행하므로, 자식 클래스는 코드를 복제하지 않고도 self.title과 self.done을 가지게 됩니다. 그런 다음 자식 클래스가 자체적으로 self.priority를 추가합니다.

PriorityTask는 mark_done()을 무료로 얻습니다—Task로부터 상속받기 때문입니다. 하지만 __str__은 우선순위를 포함하는 자체 버전으로 재정의(override)합니다. 이것이 핵심 아이디어입니다: 잘 작동하는 것은 재사용하고, 다른 부분만 변경하는 것입니다.

task = PriorityTask("Fix login bug", "high")
print(task)          # [pending] (high) Fix login bug
task.mark_done()
print(task)          # [done] (high) Fix login bug
print(task.done)     # True — inherited from Task

주의: "클래스가 항상 딕셔너리보다 낫다"는 오해

사실이 아닙니다. 딕셔너리는 더 간단하고, 빠르게 작성할 수 있으며, JSON에 직접 매핑됩니다. 데이터가 동작 없이 키-값 쌍일 뿐이라면, 딕셔너리가 올바른 도구입니다. 클래스는 데이터에 작동하는 메서드, 생성 시 검증(validation), 또는 관련 타입을 위한 상속이 필요할 때 가치를 더합니다. 작업에 맞는 도구를 사용하세요.

주의: "self는 키워드다"라는 오해

self는 키워드가 아닙니다—관례입니다. 기술적으로 this나 potato라고 이름을 지어도 Python은 불평하지 않습니다. 하지만 모든 Python 프로그래머가 self를 사용하고, 모든 AI가 self를 생성하며, 모든 린터(linter)가 self를 기대합니다. 기술적으로 선택 사항이지만, 필수로 취급하세요.

데코레이터(Decorator): 함수를 수정하는 함수

데코레이터(decorator)는 함수를 받아서 수정된 버전을 반환하는 함수입니다. @decorator 구문은 이것의 축약형입니다:

# This:
@app.route("/tasks")
def get_tasks():
    pass

# Is equivalent to:
def get_tasks():
    pass
get_tasks = app.route("/tasks")(get_tasks)

이후 챕터에서 데코레이터를 끊임없이 보게 될 것입니다 — Flask의 @app.route(), 클래스의 @staticmethod, 테스트의 @pytest.fixture. AI도 데코레이터가 적용된 함수를 지속적으로 생성합니다. 다음은 직접 만드는 방법입니다:

import time

def timer(func):
    """Decorator that prints how long a function takes."""
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        elapsed = time.time() - start
        print(f"{func.__name__} took {elapsed:.4f}s")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(0.1)
    return "done"

slow_function()  # Prints: slow_function took 0.1001s

프로덕션 데코레이터는 원래 함수의 이름과 독스트링(docstring)을 보존하기 위해 @functools.wraps(func)도 사용해야 합니다. 여기서는 간결함을 위해 생략합니다.

핵심 개념

AI가 생성한 코드는 종종 올바르게 보이지만 미묘한 버그가 있습니다. 테스트 없이는 코드가 동작한다는 것을 증명할 방법이 없습니다. 에러 처리 없이는 잘못된 입력 하나로 프로그램이 크래시됩니다. 이 챕터는 코드를—여러분의 코드든 AI의 코드든—검증하는 도구를 제공합니다.

def retry(max_attempts=3):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except:
                    if attempt == max_attempts - 1:
                        raise
        return wrapper
    return decorator
