章 05 関数、論理、制御フロー

重要な概念

関数、条件分岐、ループは、事実上あらゆるプログラムに登場する3つのパターンです。Claude Code が200行の関数を生成したとき、ロジックが正しいかどうかを判断するにはこれらのパターンを認識できなければなりません。読めないものは監督できません。

フェーズ1ではコードを読むことを学びました。今度は書きます。この章では、すべてのプログラムに登場する3つの構成要素を扱います：関数（function）（再利用可能なブロック）、条件分岐（判断ポイント）、ループ（loop）（繰り返し）。

関数

関数はキッチンのレシピカードのようなものです：名前、材料のリスト（パラメータ）、手順が書かれています。一度レシピを書けば、その料理が必要なときはいつでも使えます。コードでいえば：一度定義して、必要なときに呼び出す。一つの関数、一つの仕事。

def greet(name):
    return f"Hello, {name}!"

print(greet("World"))  # Hello, World!

def は関数を定義します。name はパラメータ—入力です。return は値を返します。関数を定義してもそれは実行されません；呼び出すことで実行されます。

比較演算子とブール演算子

プログラムが判断を下すには、はい・いいえの質問をする方法が必要です。比較演算子は2つの値を比較してブール値（True または False）を生成します：

注意：= と == の違い

= は代入（値をボックスに入れる）です。== は比較（この2つの値は等しいか？）です。if x == 5: と書くべきところを if x = 5: と書くのは、初心者が犯す最もよくあるミスの一つです—AI が生成したコードではほぼ見られません。

TaskForge では if command == "add": のような比較演算子のパターンや、for task in tasks: のように in を使ったイテレーションが見られます。これらの演算子はデータと判断をつなぐ接着剤であり、以下の条件分岐の前提条件です。

条件分岐

if/elif/else を使うと、プログラムがパスを選択できます。プログラムは最初の真のブランチを実行し、残りをスキップします。

プログラムは1つのパスを進みます。一度ブランチに入ると、他のブランチはすべてスキップされます。

ループ

for ループはコレクションの各アイテムを処理します。while ループは条件が変わるまで繰り返します。

ループは1アイテムずつ処理します。アキュムレータが実行中の結果を記憶します。

スコープ

関数の内側で定義された変数は外からは見えません。これをスコープといいます。AI が生成したコードでよくあるバグの原因です—AI は別の関数の内部にしか存在しない変数を参照することがあります。

具体的な例です。この関数はカウンタを更新しているように見えますが、実際にはしていません：

# バグ: 外側の変数を変更しない
count = 0

def increment():
    count = count + 1  # UnboundLocalError!

increment()
print(count)  # ここには到達しない

Python は関数内の count = ... を見て、count をローカル変数として扱います。しかし右辺でローカルに代入される前に count を読もうとしています—そのためエラーになります。修正方法：値を渡して結果を返す。

# 修正: 渡して、返す
count = 0

def increment(current):
    return current + 1

count = increment(count)
print(count)  # 1
注意：スコープと AI 生成コード

AI が生成したコードはグローバル状態を操作することがあります。スコープの問題を認識することで、これらのバグを発見できます。関数がパラメータでも関数内で定義されてもいない変数を読み書きしているのは危険信号です。

値としての関数：高階関数とクロージャ

Python では、関数はファーストクラスの値です — 変数に代入したり、引数として渡したり、他の関数から返したりできます。AI はこれらのパターンを常に生成します。

組み込み高階関数

map()、filter()、key 関数付きの sorted() が最も一般的です。それぞれ関数を引数にとります：

# map: 全要素に関数を適用する
names = ["alice", "bob", "charlie"]
upper_names = list(map(str.upper, names))  # ["ALICE", "BOB", "CHARLIE"]

# filter: 関数が True を返す要素のみ残す
numbers = [1, 2, 3, 4, 5, 6]
evens = list(filter(lambda x: x % 2 == 0, numbers))  # [2, 4, 6]

# key 付き sorted: カスタム基準でソートする
tasks = [{"title": "B task", "priority": 2}, {"title": "A task", "priority": 1}]
by_priority = sorted(tasks, key=lambda t: t["priority"])
重要な概念：map/filter とリスト内包表記

AI は map() と filter() を頻繁に生成します。Python では通常リスト内包表記の方が読みやすいです：list(map(str.upper, names)) より [x.upper() for x in names] の方が明快です。両方を知っていれば AI のコードを読んで、文脈に合った方を選択できます。

ラムダ式

lambda は無名関数です — 短い使い捨ての引数として便利です：

# Lambda: 一回限りの使用のための無名関数
square = lambda x: x ** 2  # def square(x): return x ** 2 と同等

# 高階関数の引数として最も便利
sorted_tasks = sorted(tasks, key=lambda t: t["due_date"])

目安：ラムダが1つ以上の式になるなら、代わりに名前付き関数を使ってください。簡潔さより可読性が重要です。

実際には、PEP 8 は名前付き関数には def の使用を推奨しています。ラムダはインライン引数として輝きます：sorted(names, key=lambda n: len(n))。

クロージャ

クロージャ（closure）は、定義されたスコープの変数を記憶する関数です：

def make_multiplier(n):
    def multiplier(x):
        return x * n  # 'n' は外側のスコープから記憶される
    return multiplier

double = make_multiplier(2)
triple = make_multiplier(3)
print(double(5))  # 10
print(triple(5))  # 15

AI はコールバック、イベントハンドラ、ファクトリパターンでクロージャを生成します。理解していれば、これらのパターンが登場したときに読んでデバッグできます。

演習：高階関数

全要素に関数を適用する apply_to_all(func, lst)（map のように）と、func が True を返す要素のみ残す keep_if(func, lst)（filter のように）を実装してください。

apply_to_all：新しいリストを作り、lst をループして各アイテムに func(item) を append します。

keep_if：新しいリストを作り、lst をループして func(item) が True のときだけ append します。

apply_to_all: return [func(x) for x in lst]。keep_if: return [x for x in lst if func(x)]。

演習：クロージャ — バリデータを作る

文字列の長さが範囲内かどうかをチェックする関数を返す make_validator(min_len, max_len) を実装してください。

文字列を受け取り min_len <= len(s) <= max_len をチェックする内部関数を返してください。

内部関数は外側のスコープから min_len と max_len を「閉じ込めます」。

def make_validator(min_len, max_len): def check(s): return min_len <= len(s) <= max_len; return check

解いて比べる：単語頻度カウンター

ステップ1：自分でこの問題を解いてください。ステップ2：「ヒント」を3回クリックしてAIの解法を見てください。ステップ3：比較してコメントに評価の質問に答えてください。

まず自分で解いてください — .lower() と .split() を使い、単語をループして辞書でカウントします。

句読点を除去するには、分割後の各単語に word.strip(".,!?;:") を使います。

AIの解法：
from collections import Counter
def word_freq(text):
    words = text.lower().split()
    cleaned = [w.strip(".,!?;:") for w in words]
    return dict(Counter(w for w in cleaned if w))

評価：AIは collections の Counter を使いました — あなたのループがすることをワンライナーでやっています。手動のアプローチも完全に機能します。しかしこれで Counter が存在することを知りました — 語彙に加えてください。TaskForge では、タスクのステータスを数えるのに使えるかもしれません。

章 06 データ構造 — リスト、辞書、データの格納方法

重要な概念

すべての API レスポンス、すべての設定ファイル、すべての AI ツール出力は構造化データです—通常は JSON で、それはネストされたリストと辞書にすぎません。ネストされたデータを操作できないと、API を使ったり、AI 出力を読んだり、データの問題をデバッグしたりできません。

関数はデータを処理します。ではデータはどこにあるのでしょうか？データ構造—情報を整理するコンテナです。Python と AI ツールで支配的な2つの構造：リスト（list）（順序付きシーケンス）と辞書（dictionary）（キーと値のペア）。

リスト

リストは 0 から始まるインデックスで管理された順序付きコレクションです：

signals = ["speed", "rpm", "fuel"]
print(signals[0])   # "speed"
print(signals[-1])  # "fuel"（最後の要素）
signals.append("temp")  # 末尾に追加

辞書

辞書はキーを値にマッピングします。JSON（ユニバーサルデータ形式）は本質的にネストされた辞書なので、API と AI ツールにとって最も重要な構造です。

warning = {"level": "critical", "system": "brakes"}
print(warning["level"])  # "critical"

ネスト構造

実際のデータはほぼ常にネストされています—辞書の中にリスト、その中に辞書：

vehicles = [
    {"name": "eCanter", "type": "EV", "range_km": 200},
    {"name": "Super Great", "type": "diesel", "range_km": 800}
]
print(vehicles[0]["name"])  # "eCanter"
リストは順序付きアイテムを保持します。辞書は名前付きアイテムを保持します。実際のデータは通常、辞書のリストを含むネストされた辞書です。

リスト内包表記

リスト内包表記は1行で新しいリストを構築します。リストに append する for ループのコンパクトな代替手段です。AI が生成する Python では内包表記を多用するので、流暢に読めるようになる必要があります。

基本構文

[expression for item in iterable]

これは「イテラブルの各 item に対して expression を評価し、結果をリストに収める」と読みます。

条件付き

[expression for item in iterable if condition]

これはフィルタを追加します：condition が真のアイテムのみが含まれます。

ループと内包表記の比較

同じロジックをループと内包表記で書いた例：

# ループバージョン
squares = []
for n in range(5):
    squares.append(n ** 2)

# 内包表記バージョン
squares = [n ** 2 for n in range(5)]
# どちらも生成するのは: [0, 1, 4, 9, 16]
# フィルタ付きループ
evens = []
for n in range(10):
    if n % 2 == 0:
        evens.append(n)

# フィルタ付き内包表記
evens = [n for n in range(10) if n % 2 == 0]
# どちらも生成するのは: [0, 2, 4, 6, 8]
注意：AI コードレビューにとって重要な理由

AI が生成したコードは内包表記を常に使います。読めなければ AI の出力をレビューできません。[task for task in tasks if task["status"] == "pending"] を見たとき、「保留中のタスクのみにタスクリストをフィルタする」と即座に認識できる必要があります。

マイクロ演習：内包表記変換

パートA：このループを内包表記に書き換えてください：

章 07 クラスとオブジェクト指向の基礎

重要な概念

AI が生成する Python コードは頻繁にクラスを使います。Claude が class Task: と self.tasks を書いたとき、あなたは何を読んでいるか理解する必要があります。

これまでデータを辞書に格納して関数で処理してきました。これは機能します—データとそれを操作する関数がコードベース全体に散らばり始めるまでは。クラスはデータと振る舞いをオブジェクトと呼ばれる単一のユニットにまとめることでこの問題を解決します。

クラスとは？

クラスはオブジェクトを作るための設計図です。クッキーの型抜きのように考えてください：クラスは形を定義し、型抜きで作る各クッキーがインスタンス—その設計図から作られた個別のオブジェクトです。クッキーの型抜き自体はクッキーではありません；それはクッキーを作るテンプレートです。すべてのクッキーは同じ形ですが、それぞれ異なるフロスティング（異なるデータ）を持てます。

プログラミング用語では：クラスはすべてのインスタンスが持つ属性（データ）とメソッド（振る舞い）を定義します。各インスタンスはデータの独自コピーを持ちますが、すべて同じメソッドを共有します。

クラスの定義

最もシンプルな便利なクラスです：

class Task:
    def __init__(self, title):
        self.title = title
        self.done = False

これを一つずつ分解してみましょう：

  • class Task: — Task という名前の新しいクラスを宣言します。クラス名は慣習として（関数のような snake_case ではなく）CamelCase を使います。
  • __init__ — イニシャライザ（コンストラクタとも呼ばれます）。Python は新しいインスタンスを作るときにこれを自動的に呼び出します。二重のアンダースコア（「ダンダー」と呼ばれます）は特殊メソッドであることを示します。
  • self — 作成される特定のインスタンスへの参照。self.title = title と書くとき、「この特定のオブジェクトのタイトルは渡されたものです」という意味です。クラスのすべてのメソッドは最初のパラメータとして self を受け取ります。
  • self.title と self.done — これらはインスタンス属性です。各 Task オブジェクトは独自の title と done を持ちます。

メソッドと関数の違い

メソッドはクラスに属する関数です。唯一の構文的な違い：メソッドは最初のパラメータとして self を取り、インスタンスのデータにアクセスできます。通常の関数は独立しています；メソッドはオブジェクトに付属し、そのオブジェクトのデータを操作します。

# 通常の関数 — 独立している
def mark_done(task_dict):
    task_dict["done"] = True

# メソッド — クラスに属し、self を操作する
class Task:
    def __init__(self, title):
        self.title = title
        self.done = False

    def mark_done(self):
        self.done = True

task.mark_done() を呼ぶとき、Python は自動的にインスタンスを self として渡します。task.mark_done(task) とは書きません—Python が処理します。

インスタンスの作成

クラスを関数のように呼び出してインスタンスを作ります：

task = Task("Buy groceries")
print(task.title)  # Buy groceries
print(task.done)   # False

"Buy groceries" は渡しますが self は渡しません。Python が自動的に self を補填します。Task(...) を呼ぶたびに新しい独立したオブジェクトが作られます。

インスタンス属性

各インスタンスは独自のデータを持ちます。一方のオブジェクトを変更しても他方には影響しません：

task1 = Task("Buy groceries")
task2 = Task("Write report")

task1.mark_done()
print(task1.done)  # True
print(task2.done)  # False — 完全に独立している

属性にはドット記法でアクセスします：task.title、task.done。これは文字列（"hello".upper()）やリスト（my_list.append(x)）で使ってきた構文と同じです—文字列やリストもオブジェクトだからです。

完全な例

イニシャライザ、振る舞いメソッド、文字列表現を持つ Task クラスです：

class Task:
    def __init__(self, title):
        self.title = title
        self.done = False

    def mark_done(self):
        self.done = True

    def __str__(self):
        status = "done" if self.done else "pending"
        return f"[{status}] {self.title}"

# 使用例
task = Task("Buy groceries")
print(task)          # [pending] Buy groceries
task.mark_done()
print(task)          # [done] Buy groceries

__str__ は別のダンダーメソッドです。オブジェクトを print() したり文字列に変換したりすると Python が自動的にこれを呼びます。これがなければ Task を表示したとき <__main__.Task object at 0x...> のような役に立たない表示になります。

クラスは3つのコンパートメントを持つボックスです：上に名前、中にデータ（属性）、下に振る舞い（メソッド）。これが UML クラス図のパターンです。

継承の基礎

既存のクラスとほぼ同じだが、いくつか追加がある新しいクラスが必要なことがあります。継承を使えば、既存のクラスをベースにした新しいクラスを作れます。新しいクラス（子クラスまたはサブクラス）は既存のクラス（親クラスまたはスーパークラス）のすべての属性とメソッドを継承し、それらに追加したり上書きしたりできます。

優先度フィールドを追加した Task を拡張した PriorityTask の例です：

class PriorityTask(Task):
    def __init__(self, title, priority="medium"):
        super().__init__(title)
        self.priority = priority

    def __str__(self):
        status = "done" if self.done else "pending"
        return f"[{status}] ({self.priority}) {self.title}"

class PriorityTask(Task) のカッコは「PriorityTask は Task を継承する」という意味です。super().__init__(title) の呼び出しは親の __init__ を実行するので、子はコードを複製することなく self.title と self.done を得ます。そして子は自身の self.priority を追加します。

PriorityTask は mark_done() を無償で得ます—Task から継承します。しかし優先度を含む独自バージョンで __str__ を上書きします。これが核心的なアイデアです：機能するものを再利用し、異なるものだけを変える。

task = PriorityTask("Fix login bug", "high")
print(task)          # [pending] (high) Fix login bug
task.mark_done()
print(task)          # [done] (high) Fix login bug
print(task.done)     # True — Task から継承

クラスと辞書：どちらを使うか

クラスと辞書はどちらもデータを格納します。問題は：データに振る舞いが必要かどうかです？

辞書とクラスのどちらを使うかのガイド
シナリオ辞書を使うクラスを使う
既知のキーを持つシンプルなデータ{"name": "Alice", "age": 30}やりすぎ
JSON との入出力自然な選択—辞書は JSON に直接マッピング必要なら to_dict() を追加
データと振る舞いをまとめて関数がファイル全体に散らばるtask.mark_done() でまとめて管理
複数の関連する型継承の仕組みがないPriorityTask(Task)
クイックプロトタイピング最も速く書けるセットアップが多く、構造が増える

目安：辞書から始めてください。同じ辞書を最初の引数として取る複数の関数を書いていることに気づいたら、データと振る舞いをクラスにまとめたいサインです。

よくある誤解

注意：「クラスは常に辞書より優れている」という誤解

そうではありません。辞書はシンプルで書くのが速く、JSON に直接マッピングされます。データがキーと値のペアで振る舞いがないなら、辞書が正しいツールです。クラスが価値を発揮するのは、データを操作するメソッド、作成時のバリデーション、関連する型の継承が必要なときです。正しいツールを選んでください。

注意：「self はキーワードである」という誤解

self はキーワードではありません—慣習です。技術的には this や potato と名付けても Python は文句を言いません。しかしすべての Python プログラマーが self を使い、すべての AI が self を生成し、すべてのリンターが self を期待します。技術的には任意ですが、必須として扱ってください。

デコレータ：関数を修正する関数

デコレータ（decorator）は関数を受け取って修正されたバージョンを返す関数です。@decorator 構文は省略記法です：

デコレータは以降の章で常に登場します — Flask の @app.route()、クラスの @staticmethod、テストの @pytest.fixture。AI はデコレータ付き関数を常に生成します。構築方法を見てみましょう：

import time

def timer(func):
    """関数にかかる時間を表示するデコレータ。"""
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        elapsed = time.time() - start
        print(f"{func.__name__} took {elapsed:.4f}s")
        return result
    return wrapper

@timer
def slow_function():
    time.sleep(0.1)
    return "done"

slow_function()  # 表示: slow_function took 0.1001s

本番のデコレータは元の関数の名前とドキュメント文字列を保持するために @functools.wraps(func) も使うべきです。ここでは簡略化のために省略します。

        self.transactions = []

        if amount > self.balance:
            raise ValueError("Insufficient funds")
        self.balance -= amount
        self.transactions.append({"type": "withdraw", "amount": amount, "time": str(datetime.now())})
        return self.balance

    def get_balance(self):
        return self.balance

    def __repr__(self):
        return f"BankAccount({self.owner}, balance={self.balance})"

評価：AIは要求していない機能を追加しました — 取引履歴、タイムスタンプ、__repr__ メソッド。これは過剰エンジニアリングで、最も一般的な AI の失敗モードの一つです。シンプルなあなたのバージョンの方がこの要件には適しています。TaskForge では「念のため」機能を追加したい衝動に抵抗してください — 仕様が求めるものを構築してください。

章 08 ファイル、モジュール、標準ライブラリ

重要な概念

実際のプロジェクトには数十のファイルがあります。Claude Code が src/models.py を編集するとき、そのファイルが src/api.py と tests/test_models.py にどう接続しているか知る必要があります。モジュールはコードが複数ファイルにまたがって整理される方法です。これがないと実際のコードベースを操作できません。

これまでプログラムが終わるとデータは消えていました。ファイルによってデータが永続します。モジュールによってコードを複数ファイルにまたがって整理できます。標準ライブラリは数百の事前構築されたツールを提供します。

ファイルの読み書き

# 書き込み
with open("data.txt", "w") as f:
    f.write("Hello, file!")

# 読み込み
with open("data.txt", "r") as f:
    content = f.read()
    print(content)  # Hello, file!

with 文（コンテキストマネージャ）は終了時に自動的にファイルを閉じます。常にこれを使ってください。

JSON：ユニバーサルデータフォーマット

JSON（JavaScript Object Notation）は AI ツールと API がデータを交換する方法です。Python の辞書とほぼ同じです：

import json

# 辞書を JSON 文字列に変換
data = {"name": "TaskForge", "version": "0.1"}
json_string = json.dumps(data, indent=2)

# JSON 文字列を辞書に変換
parsed = json.loads(json_string)

# 辞書を JSON ファイルに保存
with open("config.json", "w") as f:
    json.dump(data, f, indent=2)

# JSON ファイルを辞書に読み込む
with open("config.json", "r") as f:
    loaded = json.load(f)
異なるフォーマットは同じデータを異なる方法で格納します。JSON は AI ツールと API が使うフォーマットです。よく理解してください。

モジュールとインポート

モジュール（module）は関数を含む .py ファイルです—ツールボックスのようなものです。すべてのツールをどこにでも持ち運ぶ代わりに、現在の作業に必要なツールボックスだけを import します：

import json          # モジュール全体
from pathlib import Path  # モジュールから1つだけ

ヘッダー用にすべての辞書のユニークなキーを収集します：headers = list(data[0].keys()) はすべての辞書が同じなら機能します。

AIの解法：
def generate_report(data):
    if not data:
        return ""
    headers = list(data[0].keys())
    lines = [",".join(headers)]
    for row in data:
        lines.append(",".join(str(row[h]) for h in headers))
    return " ".join(lines)

評価：AIは最初の辞書のキーからのみヘッダーをハードコードしました。後の辞書に追加のキー（"c" など）がある場合、それらの列は黙って無視されます — そして最初の辞書にないキーを持つ行に対して row[h] は KeyError でクラッシュします。あなたのバージョンは混在するキーを処理できましたか？TaskForge では、オプションフィールドを持つタスクの CSV エクスポートでまさにこのバグに遭遇します。注意：AIの解法は実際に上記の混在キーテストに失敗します。これが要点です — 常にテストを実行してください。

章 09 エラー処理、デバッグ、テスト

重要な概念

AI が生成したコードは正しく見えても微妙なバグがあることが多いです。テストなしでは機能することを証明できません。エラー処理なしでは不正な入力の1つがプログラムをクラッシュさせます。この章ではコード—あなたのものも AI のものも—を検証するツールを提供します。

コードは壊れます。それは普通です。プロとビギナーを分けるのは壊れをどう扱い、どう防ぐかです。この章では3つのツールを扱います：try/except（エラーを穏やかに処理）、デバッグ（何が悪いかを見つける）、テスト（コードが正しいことを証明する）。

Try/Except

try:
    task_id = int(input("Task ID: "))
except ValueError:
    print("Please enter a number.")

try/except なしでは不正な入力でプログラムがクラッシュします。これで穏やかにエラーを処理できます。常に特定の例外を補足してください—裸の except: は絶対に書かないでください。

デバッグ

何かが間違っているとき、スタックトレースを下から上に読んでください。最後の行にエラーの種類とメッセージがあります。上の行にどこで起きたかが示されます。

アサーションによるテスト

アサーションは真でなければならない文です。偽なら、プログラムはクラッシュします—これはテストでまさに欲しい動作です。

assert 2 + 2 == 4      # 静かに通過する
assert 2 + 2 == 5      # AssertionError!
AI に依頼する前にテストを書いてください。テストが契約を定義します。AI はそれに基づいて実装します。
重要な概念：テスト駆動型 AI 開発

AI にコードを書かせる前にテストを書いてください。これは最も高いレバレッジを持つ AI コーディングワークフローです。あなたのテストが「正解」を定義します。AI はそれに通るコードを書きます。失敗したら繰り返します。

注意：テストなしで何が起きるか

AI に calculate_tax(income) を書かせました。正しく見える関数を返します。デプロイします。1ヶ月後、正しいブラケットベースの率ではなく 15% で計算していたことを発見します—すべての請求書が間違っていました。テストがあれば秒で発見できたはずです。テストなしでは AI の出力を信仰で信頼しています。テストがあれば証拠があります。

デザインチャレンジ：「リトライデコレータ」

fn() を呼び出す retry(fn, max_attempts=3) 関数を書いてください。例外が発生した場合、max_attempts 回までリトライします。成功した場合は結果を返し、すべての試みが失敗した場合は最後の例外を再発生させます。

for ループを使います：for attempt in range(max_attempts)

ループの内側で fn() を try/except でラップします。成功したら即座に return します。

ループ後、最後の試みの except ブロックの内側で再発生させます：raise

解いて比べる：リトライデコレータ

ステップ1：自分でこの問題を解いてください。ステップ2：「ヒント」を3回クリックしてAIのバージョンを見てください。ステップ3：AIのコードに潜む本物のバグを探してください。

retry(max_attempts) はデコレータを返します。そのデコレータは関数を受け取りラッパーを返します。3段階のネストです。

ラッパーは range(max_attempts) をループし、try/except で func() を呼びます。成功したら return。最後の失敗で再発生させます。

AIの解法：
def retry(max_attempts=3):
    def decorator(func):
        def wrapper(*args, **kwargs):
            for attempt in range(max_attempts):
                try:
                    return func(*args, **kwargs)
                except:
                    if attempt == max_attempts - 1:
                        raise
        return wrapper
    return decorator

評価：AIは except Exception: の代わりに裸の except: を使いました。これは KeyboardInterrupt と SystemExit も補足するので、リトライ中に Ctrl+C でプログラムを止められなくなります。これはエラー処理を理解している人だけが発見できる微妙なバグです。TaskForge では裸の except を持つリトライデコレータは CLI ツールを終了不可能にする恐れがあります。
