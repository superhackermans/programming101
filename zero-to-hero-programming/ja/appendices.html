import json
from pathlib import Path

def load_data(filepath: str) -> list[dict]:
    """ファイルからJSONデータを読み込む。ファイルがなければ空リストを返す。"""
    path = Path(filepath)
    if not path.exists():
        return []
    return json.loads(path.read_text())

def save_data(filepath: str, data: list[dict]) -> None:
    """データを整形されたJSONファイルに書き出す。"""
    Path(filepath).write_text(json.dumps(data, indent=2))

# 使用例
tasks = load_data("tasks.json")
tasks.append({"id": len(tasks) + 1, "title": "新しいタスク", "status": "todo"})
save_data("tasks.json", tasks)

Ch 14: Flask APIエンドポイント

from flask import Flask, jsonify, request

app = Flask(__name__)

tasks = []

@app.route("/api/tasks", methods=["GET"])
def get_tasks():
    """すべてのタスクを返す。ステータスでフィルタリング可能。"""
    status = request.args.get("status")
    if status:
        filtered = [t for t in tasks if t["status"] == status]
        return jsonify(filtered)
    return jsonify(tasks)

@app.route("/api/tasks", methods=["POST"])
def create_task():
    """JSONボディから新しいタスクを作成する。"""
    data = request.get_json()
    if not data or "title" not in data:
        return jsonify({"error": "titleは必須です"}), 400
    task = {
        "id": len(tasks) + 1,
        "title": data["title"],
        "status": data.get("status", "todo"),
    }
    tasks.append(task)
    return jsonify(task), 201

if __name__ == "__main__":
    app.run(debug=True)

フェーズ1の問題

Pythonが見つからない
▶
症状
ターミナルでpython3: command not foundまたは'python3' is not recognizedと表示される。
原因
Pythonがインストールされていないか、インストールディレクトリがシステムのPATHに含まれていません。
修正
python.orgからPythonを再インストールし、「Add to PATH」にチェックを入れてください。macOSではbrew install python3を実行します。which python3で確認してください。
「command not found」エラー
▶
症状
インストール後にツール（git、node、code）が「command not found」を返す。
原因
実行ファイルのディレクトリがシェルのPATHに含まれていないか、シェルセッションが再起動されていません。
修正
ターミナルを閉じて再度開いてください。それでも解決しない場合は、インストールパスを~/.zshrcまたは~/.bashrcに追加し、source ~/.zshrcを実行してください。
REPLが起動しない
▶
症状
python3と入力するとハングするか、予期しないバージョンが表示される。
原因
複数のPythonインストールが競合しているか、エイリアスが誤ったバージョンを指しています。
修正
which python3を実行して、どのバイナリが使われるか確認してください。フルパス（例：/usr/local/bin/python3）を使うか、シェル設定でPATHの順序を修正してください。

▶
症状

コマンドパレット（Cmd+Shift+P）を開き、「Python: Select Interpreter」と入力して、正しいPython 3のインストールを選択してください。

ファイルでタブとスペースが混在してインデントされているか、インデントレベルが一貫していません。
修正
エディターをスペース4つ（タブ不使用）に設定してください。VS Codeでは：設定 → 「Editor: Tab Size」= 4、「Editor: Insert Spaces」= チェック済み。影響を受けたファイルを再インデントしてください。
ModuleNotFoundError（モジュール未検出エラー）
▶
症状
ModuleNotFoundError: No module named 'requests'（または任意のサードパーティパッケージ）。
原因
使用しているPython環境にパッケージがインストールされていないか、複数のPythonがインストールされています。
修正
python3 -m pip install requestsを実行してください（python3 -m pipを使うと正しいPythonにインストールされます）。仮想環境を使って依存関係を分離してください。
json.decoder.JSONDecodeError
▶
症状
JSONファイルの読み取り時にjson.decoder.JSONDecodeError: Expecting value。
原因
ファイルが空か、無効なJSON（末尾のカンマや二重引用符の代わりに単一引用符など）が含まれています。
修正
jsonlint.comでJSONを検証してください。ファイルが空でないことを確認してください。手動編集の代わりにjson.dumps()でJSONを書き出してください。
pytestが見つからない
▶
症状
python3 -m pytestの実行時にNo module named pytest。
原因
現在のPython環境にpytestがインストールされていません。
修正
python3 -m pip install pytestを実行してください。仮想環境を使用している場合は、先に有効化してください（source venv/bin/activate）。

フェーズ3の問題

Permission denied（ファイル/SSH）
▶
症状
ファイルの読み取りやSSH経由のgitリモートへの接続時にPermission denied。
原因
ファイルのパーミッションが制限的すぎるか、SSHキーがSSHエージェントまたはGitHubに追加されていません。
修正
ファイルの場合：chmod 644 ファイル名。SSHの場合：ssh-add ~/.ssh/id_ed25519を実行し、GitHubの設定 → SSH Keysでキーが追加されているか確認してください。
Gitの「detached HEAD」
▶
症状
GitがHEAD detached at abc1234と表示し、コミットが消えたように見える。
原因
ブランチの代わりに特定のコミットまたはタグをチェックアウトしたため、新しいコミットを追跡するブランチがありません。
修正
現在の位置からブランチを作成してください：git checkout -b my-branch。名前付きブランチに乗ればコミットは安全です。
マージコンフリクト
▶
症状
ファイルに<<<<<<<マーカーが表示され、gitがマージを完了しない。
原因
2つのブランチが同じ行を編集しました。Gitはどちらのバージョンを保持するか自動的に決定できません。
修正
コンフリクトしたファイルを開き、マーカー間の正しいコードを選択し、<<<<<<< / ======= / >>>>>>>の行をすべて削除してからgit addとgit commitを実行してください。

フェーズ4・5の問題

「ポートがすでに使用中」
▶
症状
Flaskや他のサーバー起動時にOSError: [Errno 48] Address already in use。
原因
他のプロセスがすでにそのポートを使用しています（停止し忘れた以前のサーバーなど）。
修正
プロセスを検索：lsof -i :5000。強制終了：kill -9 <PID>。または別のポートでサーバーを起動：flask run --port 5001。
Claude Codeがインストールできない
▶
症状
npm install -g @anthropic-ai/claude-codeがエンジンまたはバージョンエラーで失敗する。
原因
Node.jsのバージョンが古すぎます。Claude CodeはNode.js 18以降が必要です。
修正
node --versionで確認してください。nodejs.orgから最新のLTSをインストールするか、nvm install --ltsを使用してください。
CLAUDE.mdが読み込まれない
▶
症状
Claude CodeがCLAUDE.mdのルールを無視する。動作が指示と一致しない。
原因
CLAUDE.mdファイルがプロジェクトのルートディレクトリにないか、別のディレクトリからClaude Codeを起動しています。
修正
pwdで現在のディレクトリを確認してください。CLAUDE.mdがclaudeを実行するディレクトリと同じ場所にあることを確認してください。ls CLAUDE.mdで確認できます。
APIキーエラー
▶
症状
Claude Code起動時にAuthenticationErrorまたは「invalid API key」。
原因
APIキーが見つからない、期限切れ、または誤った環境変数に設定されています。
修正
claudeを実行して認証プロンプトに従ってください。APIキーを直接使用する場合は、シェル設定でANTHROPIC_API_KEYが正しく設定されているか確認してください。
MCPサーバーへの接続失敗
▶
症状
Claude Codeが「MCP server failed to connect」と報告するか、ツールが利用できない。
原因
MCPサーバープロセスがクラッシュした、設定ファイルにタイポがある、または必要な依存関係が不足しています。
修正
.claude/mcp.jsonの構文エラーを確認してください。MCPサーバーコマンドを手動で実行してエラー出力を確認してください。すべての依存関係がインストールされていることを確認してください。
レート制限エラー
▶
症状
Claude Code使用時に429 Too Many RequestsまたはRateLimitError。
原因
APIのレート制限またはプランの使用量上限を超えています。
修正
60秒待ってから再試行してください。継続する場合はconsole.anthropic.comで使用量を確認してください。Claude MaxサブスクライバーはAPIキーユーザーより高い制限があります。演習中に制限に達した場合は新しい会話を開始してください — コンテキストが短いほどトークン消費が少なくなります。
ネットワーク / ファイアウォールによるブロック
▶
症状
Claude Codeが接続しようとするとECONNREFUSED、ETIMEDOUT、またはfetch failed。
原因
ネットワークがapi.anthropic.comへのアウトバウンドHTTPS接続をブロックしているか、企業のファイアウォール/VPN内にいます。
修正
curl https://api.anthropic.com/v1/messages -Iで接続を確認してください。ブロックされている場合は、ファイアウォールの設定を確認するか、別のネットワークから試してください。企業のプロキシを使用している場合はHTTPS_PROXY環境変数を設定してください：export HTTPS_PROXY=http://your-proxy:port
プロキシ設定
▶
症状
Claude CodeはインストールできるがAPIに接続できない。npm install自体がネットワークエラーで失敗する。
原因
npmとClaude Codeはプロキシ設定を別々に必要とします。
修正
npmの場合：npm config set proxy http://your-proxy:portとnpm config set https-proxy http://your-proxy:port。Claude Codeの場合：シェル設定ファイル（~/.zshrcまたは~/.bashrc）にHTTPS_PROXYを設定し、source ~/.zshrcを実行してください。

フェーズ9の問題

Dispatchが見つからない
▶
症状
dispatch: command not foundまたは/dispatchコマンドが認識されない。
原因
Dispatchツールがインストールされていないか、使用しているClaude Codeのバージョンで利用できません。
修正
Claude Codeを最新バージョンに更新してください：npm update -g @anthropic-ai/claude-code。プランの機能セットでDispatchの利用可否を確認してください。
バックグラウンドエージェントがサイレントに失敗する
▶
症状
ディスパッチされたタスクが「完了」と表示されるが、出力がないか間違った結果を生成する。
原因
エージェントのコンテキストが不十分か、ハーネス（テスト/リンター）が失敗を検出できませんでした。
修正
エージェントの出力ログでエラーを確認してください。ハーネスに十分なテストカバレッジがあることを確認してください。失敗が明示的に表面化するよう、ディスパッチプロンプトに制約を追加してください。
並行エージェントからのマージコンフリクト
▶
症状
複数のエージェントブランチがmainにマージするときにコンフリクトする。
原因
2つ以上のエージェントが同じファイルを変更しました。並行タスクをディスパッチする際には想定内の動作です。
修正
各エージェントを分離するためにワークツリーを使用してください。可能であればエージェントに重複しないファイルを割り当ててください。1ブランチずつマージしてコンフリクトを逐次解決してください。
Dockerのpermission denied
▶
症状
permission denied while trying to connect to the Docker daemon socket。
原因
ユーザーがdockerグループに所属していないか、Docker Desktopが実行されていません。
修正
macOSの場合：Docker Desktopを起動してください。Linuxの場合：sudo usermod -aG docker $USERを実行してログアウト/ログインしてください。docker psで確認できます。

フェーズ10の問題

エージェントチームが競合するファイル変更を生成する
▶
症状
2つのエージェントが同じファイルを同時に編集し、一方が他方の変更を上書きする。
原因
並行エージェント間にファイルロックや連携がありません。
修正
Ch 46のハブアンドスポークパターンを使用してください。各エージェントに別々のファイルまたはセクションを割り当ててください。エージェントがファイルを共有しなければならない場合は、オーケストレーターを通じて書き込みを直列化してください。
バックグラウンドエージェントのタイムアウトまたはハング
▶
症状
バックグラウンドエージェントが出力を停止し、完了しない。
原因
エージェントが無限再試行ループに入った、レート制限に達した、または受け取れないユーザー入力を待っています。
修正
エージェント設定にmax_iterationsを設定してください。エージェントの出力ログを監視してください。詰まった場合はプロセスを停止し、より狭いタスクで新しいエージェントをディスパッチしてください。

体系的な読書パス

開始タイミング書籍得られるもの読み方
フェーズ2修了後 Composing Programs（無料オンライン） 関数、データ抽象化、再帰、インタープリタの深い理解。 第1〜2章を読み、すべての演習をこなす。

AIとの関連性

このリストの書籍はすべて、AIの監督者として優れたプログラマーになるための学習に役立ちます。Composing ProgramsはAIが生成するコードを読む力を養います。The Algorithm Design Manualはその効率を評価する力を養います。DDIAはそのアーキテクチャを批判的に検討する力を養います。理解が深まるほど、判断力が高まります。
