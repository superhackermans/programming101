第01章 プログラミングとは本当に何か

重要な概念

AIが生成したコードが良いか悪いかを判断するには、まずコードが何であるかを知る必要があります。この章では、カリキュラム全体を理解するための基本的なメンタルモデルを身につけます。

プログラミングとは、常識がゼロの機械に指示を書くことです。ピーナッツバターのサンドイッチを作るよう誰かに頼む場面を想像してみてください。人間なら瓶を手に取り、蓋を開け、ナイフを持ちます。コンピュータは？「蓋を開ける」と言わなかったので、密封された瓶を前にして止まったままです。すべての指示は、明示的・文字通り・順序正しく書かなければなりません。

プログラム（program）はこれらの指示を含むテキストファイルです。謎めいたバイナリの塊ではなく、どのエディタでも開ける普通のテキストファイルです。インタープリタ（interpreter）（Pythonなどの言語用）やコンパイラ（compiler）（Rustなどの言語用）が、人間が読めるテキストをプロセッサが実行できる命令に変換します。

なぜコードが存在するのか

コンピュータが理解できるのは電圧パターンだけです。高電圧 = 1、低電圧 = 0。それだけです。プログラミング言語は、人間が0と1で考えなくて済むように存在します。あなたが書くPythonコードとCPU内で切り替わる電圧の間には、いくつかの翻訳レイヤーがあり、下位のレイヤーを直接触る必要はありません。

あなたはPythonを書きます。4つの翻訳レイヤーを経て、CPUがスイッチを切り替えます。下位のレイヤーを直接触る必要はありません。

プログラミング言語の全体像

用途によってプログラミング言語が異なります。ネジにはドライバー、釘にはハンマーを使うのと同じです。主なカテゴリは以下の通りです。

用途ごとに異なる言語を使います。Pythonがあなたの出発点です。AI、スクリプティング、Webバックエンドをカバーします。

ソフトウェアの実際の姿

上の図は言語のカテゴリを示しています。では、それらで実際に何が作られているのでしょうか？ほとんどのソフトウェアはいくつかの主要なカテゴリに分類され、実際の製品はそれらを組み合わせています。

Webシステム（フロントエンド + バックエンド）

Webアプリケーション（web application）とは、ブラウザで使うすべてのもの——Gmail、YouTube、GitHub、オンラインバンキング——のことです。2つの部分から構成されています。

  • フロントエンド（frontend） — あなたが見る部分。HTMLがページの構造を作り、CSSがスタイルを適用し、JavaScriptが操作可能にします。ボタンをクリックしてページを再読み込みせずにメニューが現れる場合、それはブラウザ上でJavaScriptが動いています。
  • バックエンド（backend） — 見えない部分。サーバー（Python、Node.js、Go、Rustなどで書かれることが多い）がリクエストを処理し、ビジネスロジックを適用し、データベースと通信します。ログインするとき、バックエンドがパスワードを確認します。コメントを投稿するとき、バックエンドがそれを保存します。

フロントエンドとバックエンドはHTTPを通じて通信します——第13章で学ぶプロトコルです。あなたが訪問するすべてのURLは、どこかのバックエンドへのHTTPリクエストを引き起こします。

実際の例：Twitter/X。フロントエンドはブラウザでタイムラインを表示します。バックエンドは認証を処理し、ツイートをデータベースに保存し、レコメンデーションアルゴリズムを実行し、タイムラインデータをHTTP経由でJSONとして配信します。数百万のユーザー、数千のバックエンドサーバー、一つのシームレスな体験。

モバイルアプリケーション

スマートフォン上のアプリです。2つのアプローチがあります。

  • ネイティブアプリ（native apps） — 特定のプラットフォーム専用に作られます。iOSはSwift、AndroidはKotlinを使います。最高のパフォーマンスを持ち、デバイスのハードウェア（カメラ、GPS、センサー）に完全にアクセスできます。Instagram、iOSの計算機、Apple Healthはネイティブです。
  • クロスプラットフォームアプリ（cross-platform apps） — 一つのコードベースで両方のプラットフォームに対応します。React Native（JavaScript）やFlutter（Dart）が同じコードからiOSとAndroidのバージョンを生成します。開発が速い反面、少し完成度が落ちます。多くのスタートアップがこれを使います。

ほとんどのモバイルアプリもバックエンドが必要です——HTTP経由で同じ種類のサーバーと通信する、異なるフロントエンドに過ぎません。

データベース

データベース（database）は、整理された永続的なストレージです。データベースがなければ、プログラムが終了するとデータが消えてしまいます。主に2つの系統があります。

  • リレーショナル型（SQL） — 行と列を持つテーブルにデータが格納されます。厳格なルールを持つスプレッドシートのようなものです。PostgreSQL、MySQL、SQLiteがあります。データが明確な関係を持つ場合（ユーザーが注文を持ち、注文が商品を持つ）に最適です。第15章でSQLiteを使います。
  • 非リレーショナル型（NoSQL） — 柔軟な構造：ドキュメント（MongoDB）、キーバリューペア（Redis）、グラフ（Neo4j）など。非構造化データ、キャッシング、テーブルにきれいに収まらない関係に最適です。

実際の例：Shopifyのようなeコマースサイトは、商品・注文・顧客（構造化されたリレーショナルデータ）にリレーショナルデータベースを使います。商品ページを瞬時に読み込むためのキャッシングにRedis（NoSQL）を、商品検索にElasticsearchを使うこともあります。

それらの組み合わせ方

現代のアプリケーションは通常、これら3つを組み合わせています。Uberのような配車アプリを考えてみましょう。

  • モバイルアプリ（フロントエンド）— 地図を表示し、乗車を依頼し、ドライバーの位置をリアルタイムに表示する
  • バックエンドサーバー — 乗客とドライバーをマッチングし、料金を計算し、支払いを処理し、サージ料金のロジックを処理する
  • データベース — ユーザープロフィール、乗車履歴、支払い方法、ドライバーの評価を保存する
  • Webダッシュボード — ドライバーがアカウントを管理し、管理者がシステムを監視できるようにする

このカリキュラムはPythonのバックエンドとSQLiteデータベースに焦点を当てています。それが基礎だからです。バックエンドがリクエストを処理してデータを保存する方法を理解すれば、フロントエンド・モバイルアプリ・クラウドインフラなど、すべては同じパターンの変形に過ぎないとわかります。

よくある誤解

注意：「数学が得意でないといけない」という誤解

プログラミングは方程式を解くよりも、レシピを書くことに近いです。ほとんどのコードは情報の整理とプロセスの記述であり、微積分ではありません。数学が重要な作業（機械学習、物理シミュレーション）は専門分野であり、前提条件ではありません。

注意：「コードは難解なもの」という誤解

プログラムはテキストファイルです。このコースで作るtaskforge.pyをメモ帳で開いて、すべての行を読むことができます。隠されたものは何もありません。後でAIがコードを生成するとき、これを思い出すことで冷静でいられます——それはあなたが読んで判断できる、ただのテキストです。

以下の演習は、Pyodide（Webページ内で動くようコンパイルされたPython）を使ってブラウザ上でPythonを直接実行します。追加のインストールは不要です。今すぐ試してみましょう。

演習が読み込まれない場合

実行をクリックしても何も起きない、または「Failed to load Python runtime」と表示される場合、ブラウザがPyodideの読み込みをブロックしています。これは、HTMLファイルを直接開いたとき（アドレスバーにfile:///と表示される場合）に一部のブラウザで発生します。解決策は、ローカルWebサーバー——あなたのコンピュータで動作し、実際のWebサイトと同じようにブラウザにファイルを渡す小さなプログラム——を通じてファイルを提供することです。複雑に聞こえますが、ターミナルコマンド1つで済みます。以下の手順に従えば、ガイド全体のすべての演習が動作します。

演習が正常に動作した場合は、このセクションをスキップして次の演習に進んでください。

ローカルサーバーの起動方法（クリックして展開）

ステップ1：ターミナルを開く

  • Mac：Cmd + Spaceを押し、Terminalと入力してEnterを押します。
  • Windows：Win + Rを押し、cmdと入力してEnterを押します。（スタートメニューで「コマンドプロンプト」を検索してもOK）
  • Linux：Ctrl + Alt + Tを押します。

ステップ2：ファイルが入っているフォルダに移動する

HTMLファイルがどこにあるかをターミナルに伝える必要があります。cd（ディレクトリ変更）コマンドの後にsrcフォルダへのパスを続けます。例えば：

Mac / Linuxcd ~/Desktop/zero-to-hero-programming/src
Windowscd C:\Users\YourName\Desktop\zero-to-hero-programming\src

パスは実際にファイルを保存した場所に合わせて変更してください。わからない場合は、ファイルエクスプローラーでsrcフォルダを見つけて：

  • Mac：srcフォルダを右クリックし、Optionを押したまま「‘src’のパス名をコピー」をクリック。ターミナルでcd と入力してペーストします。
  • Windows：ファイルエクスプローラーのアドレスバーをクリックしてパスをコピー。ターミナルでcd と入力してペーストします。

ステップ3：サーバーを起動する

このコマンドを入力してEnterを押します：

ターミナルpython3 -m http.server 8000

各部分が何を意味するか説明します。ここに魔法はありません。

  • python3 — 第01章でインストールしたPythonプログラムを実行します。後でコードを書くのと同じPythonです。
  • -m — このフラグはPythonに「名前で組み込みモジュール（Pythonにプリインストールされたミニプログラム）を実行する」よう指示します。追加のダウンロードは不要です。-mは「module（モジュール）を実行」と覚えてください。
  • http.server — 組み込みモジュールの名前です。あなたのコンピュータを小さなWebサーバーに変えます。サーバーとは、リクエストを待ってファイルを返すだけのプログラムです——どのWebサイトにアクセスしても起きることと同じです。違いは、このサーバーがあなたのコンピュータで動き、あなただけがアクセスできることです。
  • 8000 — ポート番号です。あなたのコンピュータを何千もの番号付きドアを持つビルだと思ってください。ポート8000は、サーバーに待機するよう指示するドアです。8000という数字はローカル開発の慣習であり、ほぼ任意の番号でも構いませんが、ほとんどのチュートリアルが8000を使うので、それに合わせます。

表示されるもの：Enterを押すと、ターミナルに以下のような内容が表示されます：

出力Serving HTTP on 0.0.0.0 port 8000 (http://0.0.0.0:8000/) ...

これはサーバーが実行中で待機していることを意味します。ターミナルが「フリーズ」しているように見えますが、正常です。フリーズしているのではなく、リクエストを待機しています。サーバーが動いている間は、そのターミナルウィンドウでコマンドを入力できません。（これは想定通りです。後でCtrl + Cで停止します。）

python3が動かない場合：

  • Windows：代わりにpython -m http.server 8000を試してください（Windowsでは3なしでpythonとしてインストールされることがあります）。
  • 「command not found」エラー：Pythonがまだインストールされていないか、システムのPATHにありません。この章の最初のPythonインストール手順に戻って再試行してください。Macではpython3 --versionでPythonがインストールされているか確認できます。
  • 「Address already in use」エラー：別のプログラムがポート8000を使っています。別のポート番号を試してください：python3 -m http.server 8080。別のポートを使う場合は、ステップ4の8000をそのポート番号に変えてください。

ステップ4：ブラウザで開く

ターミナルは実行したままにします（サーバーは起動し続ける必要があります）。Webブラウザ（Chrome、Firefox、Safari、Edgeのどれでも可）を開き、アドレスバー（URLを入力する一番上のバー、検索バーではありません）に以下を入力します：

ブラウザアドレスバーhttp://localhost:8000

これも分解して説明します：

  • http:// — HTTPプロトコル（すべてのWebサイトが使うもの）を使うようブラウザに指示します。
  • localhost — 「このコンピュータ」を意味する特別な名前です。インターネットに出るのではなく、ブラウザはあなたのマシンで起動したサーバーと通信します。
  • :8000 — ステップ3のポート番号です。ブラウザにどの「ドア」をノックするか伝えます。

表示されるもの：srcフォルダ内のすべてのファイル（phase-1.html、phase-2.htmlなど）が一覧表示されたシンプルなページが表示されます。phase-1.htmlをクリックしてこのページに戻ると——インタラクティブ演習が動作するようになります。

動作していることを確認する方法

ブラウザのアドレスバーを確認します。http://localhost:8000/phase-1.htmlと表示されているはずです。まだfile:///...と表示されている場合、サーバー経由ではなく直接ファイルを開いたことになります——ステップ4に戻り、localhostアドレスを入力してください。

終了するとき

サーバーが動いているターミナルウィンドウに戻り、Ctrl + C（Controlキーを押しながらCを押す）を押します。これはサーバーへの「停止」シグナルを送ります。ターミナルが通常の状態に戻り、新しいコマンドを受け付けられるようになります。サーバーが停止するとブラウザでの演習は動作しなくなります——これは想定通りです。

次回演習を使いたいときは、ステップ2〜4を繰り返してください。数回やれば10秒程度で済みます。

第02章 コンピュータがコードを実行する仕組み

重要な概念

Claude Codeが関数を生成したとき、頭の中でトレースする必要があります。何が入って、何が変わり、何が出てくるのか。この章では、機械がどのように考えるかを——文字通り、一歩一歩——学びます。

プログラムが命令を含むテキストファイルだとわかったところで、機械が実際にどのようにその命令を実行するかを見てみましょう。重要な概念：逐次実行（sequential execution）——コンピュータはコードを上から下へ、あなたが書いた通りの順序で、一行ずつ読み取ります。

変数：ラベル付きの箱

変数（variable）は値を保持するラベル付きの箱です。アパートの郵便受けの列を想像してください。それぞれに名前が付いており、ラベルを変えずに中身を入れ替えることができます。Pythonでは、=記号で作成します。

# これはPythonです — このガイドを通して使う言語です
speed = 85
name = "TaskForge"
running = True

ラベル（speed）は変わりません。中の値は置き換えることができます。speed = 90は同じ箱に新しい値を入れ——古い85は消えます。

変数はラベル付きの箱です。代入は箱に新しい値を入れます。古い値は消えます。

データ型：箱の形

値の型（type）によって、有効な操作が決まります。

箱に値が入ったところで、それらを組み合わせる必要があります。式（expression）とは、値を生成するコードの任意の部分です。2 + 3は5を生成する式です。

エラーはフィードバック

コードが間違っているとき、コンピュータは教えてくれます——聞き方を知っていれば。エラーには3つの種類があります。

AIが生成したコードは構文エラーや実行時エラーをほとんど起こしません。ロジックエラー——動くけれど間違ったことをするコード——がAIが失敗する場所であり、あなたのレビューが重要な場所です。
AIにとっての重要な意味

Claude CodeなどのAIツールがコードを生成するとき、構文エラー（タイプ1）や実行時クラッシュ（タイプ2）はほとんど起きません。危険なのはロジックエラー——完全に動くのに間違った結果を生成するコード——です。だからこそ、あなたがコードが何をすべきかを理解し、AIが間違えた箇所を見つけられる必要があります。

ターミナル

ターミナルはコンピュータへのテキストベースのインターフェースです。ファイル → 新規フォルダをクリックする代わりに、mkdir new_folderと入力します。速く、スクリプト化でき、そして決定的に——すべてのAIコーディングツールが動作する方法です。Claude Codeはすべてターミナルで動作します。

ターミナルエミュレータのアップグレード

重要な概念：ターミナルエミュレータ

ターミナルエミュレータ（terminal emulator）はシェルへのウィンドウを提供します。シェルはコマンドを解釈するプログラム（bash、zsh）であり、ターミナルエミュレータはシェルを表示して入力を処理するアプリケーションです。すべてのOSにデフォルトのものが付属していますが、サードパーティのターミナルエミュレータは分割ペイン、検索、インスタントリプレイなどの機能を追加し、毎日何時間もターミナルで作業するようになると大きな違いをもたらします。

現在のツール（2026年3月）

Windows：Windows Terminal（Windows 11にプリインストール、Windows 10ではMicrosoft Storeから無料）。タブ、分割ペイン（Alt + Shift + D）、プロファイルをサポートします。これが現代のデフォルトです——レガシーのcmd.exeは使わないでください。

仮想環境（virtual environment）

Pythonパッケージをインストールする前に、仮想環境を理解する必要があります。仮想環境とは、特定のプロジェクト専用のPythonの自己完結型コピーです。仮想環境がないと、インストールしたすべてのパッケージが一つのグローバルな場所に入ります——やがて2つのプロジェクトが同じパッケージの異なるバージョンを必要とし、問題が起きます。仮想環境はこの問題を完全に防ぎます。

注意：pip installの前に必ずvenvを作成しましょう

仮想環境の外でpip installを実行してはいけません。パッケージをグローバルにインストールすると、パッケージが積み重なり、競合し、やがて診断が難しい形で問題が発生します。仮想環境は各プロジェクトのパッケージを隔離してくれます。

仮想環境の作成

ターミナル# まずプロジェクトディレクトリに移動します cd ~/my-project # .venvという名前の仮想環境を作成します python3 -m venv .venv

これにより、プロジェクト内に.venvフォルダが作成されます。Pythonのプライベートコピーと、空のパッケージセットが含まれています。フォルダ名.venvは慣習です——ドットで始まるため隠しフォルダになり、ファイル一覧を散らかしません。

仮想環境の有効化

macOS / Linuxsource .venv/bin/activate
Windows.venv\Scripts\activate

仮想環境が有効になっているかの確認方法

2つの手がかりで確認できます。

# 1. プロンプトが変わります — 先頭に(.venv)が表示されます：
(.venv) ~/my-project $

# 2. which pythonがグローバルではなくvenv内のPythonを指します：
(.venv) ~/my-project $ which python
/Users/you/my-project/.venv/bin/python

無効化

プロジェクトの作業が終わったら、deactivateと入力してグローバルのPythonに戻ります。

ターミナルdeactivate

プロンプトから(.venv)のプレフィックスが消え、which pythonがシステムのPythonを指すようになります。

第04章 書く前にコードを読む

重要な概念

AIコーディングツールの時代では、書くよりも読むコードの方がはるかに多くなります。Claude Codeが200行を生成したとき、あなたの仕事はそれを評価すること—タイプすることではありません。この章では、受動的な受け入れ者ではなく、有能なAIの監督者になるためのスキルを身につけます。

ここにパラダイムシフト（paradigm shift）があります。従来の教育はコードを書くことから始まります。AI支援の作業はコードを読むことから始まります。Claude Codeがあなたのために200行のPythonを生成したとき、その行を書くのではなく—読み、評価し、受け入れるかどうかを判断します。コードを読むことがあなたの主要なスキルです。

ロジックパス

TaskForge v0.1 — コード

以下がTaskForge v0.1の完全なソースコードです。これはコース全体を通して構築していく背骨プロジェクト（spine project）です。3パスアプローチを使って今読みましょう。

# taskforge.py — v0.1
# A minimal command-line task manager

tasks = []

def add_task(description):
    task = {"id": len(tasks) + 1, "description": description, "status": "pending"}
    tasks.append(task)
    return task

def complete_task(task_id):
    for task in tasks:
        if task["id"] == task_id:
            task["status"] = "complete"
            return task
    return None

def list_tasks():
    return tasks

def main():
    while True:
        command = input("\nTaskForge> ").strip().lower()
        if command == "add":
            desc = input("Description: ")
            task = add_task(desc)
            print(f"Added: #{task['id']} — {task['description']}")
        elif command == "done":
            task_id = int(input("Task ID: "))
            result = complete_task(task_id)
            if result:
                print(f"Completed: #{result['id']}")
            else:
                print("Task not found.")
        elif command == "list":
            for t in list_tasks():
                status = "✓" if t["status"] == "complete" else " "
                print(f"  [{status}] #{t['id']} — {t['description']}")
        elif command == "quit":
            break
        else:
            print("Commands: add, done, list, quit")

if __name__ == "__main__":
    main()
補足

while True:は無限ループ（infinite loop）を作成し、ユーザーが"quit"と入力したとき（breakが発動したとき）にのみ停止します。whileループについては第5章で正式に学びますが、パターンはシンプルです: breakが発生するまでこのブロックを永遠に繰り返します。今は形を認識するだけで十分です—先頭にwhile True:、内部のどこかにbreak。

3パスリーディングの実例

TaskForgeを自分で分析する前に、短い関数に3パス手法を適用した例を見てみましょう。各パスがどのように異なる理解の層を明らかにするか観察してください。

def find_max(numbers):
    if len(numbers) == 0:
        return None
    biggest = numbers[0]
    for num in numbers:
        if num > biggest:
            biggest = num
    return biggest
1
構造パス

find_maxという関数が1つ。パラメータは1つ: numbers。内部にループあり。何かを返す。全8行。

2
ロジックパス

リストが空ならNoneを返す。そうでなければ、最初の要素が最大だと仮定して開始。すべての数値を走査し—現在の最大値より大きい数値があれば、最大値を更新。すべての数値をチェックした後、見つかった最大値を返す。

3
エッジパス

空のリスト: Noneを返す（明示的に処理済み）。要素1つのリスト: 正常に動作—ループは1回実行され、biggestはその唯一の要素のまま。すべて同じ値: 正常に動作—num > biggestがTrueになることはなく、最初の値が返される。負の数: 正常に動作—正の値を前提にしていない。数値以外の入力（例: 文字列）: 型が混在していると>の比較でクラッシュする。入力型のバリデーション（validation）なし。
