코드 경험 제로에서 AI 에이전트 오케스트레이션까지

코드를 한 줄도 작성해본 적 없는 분도 괜찮습니다. 이 인터랙티브 가이드는 프로그래밍의 기초부터 멀티 에이전트 AI 코딩 시스템을 직접 감독하는 수준까지 단계별로 안내합니다. 각 단계에서는 다음 단계를 위한 판단력을 키워드리며, 불필요한 내용은 하나도 없습니다—모든 챕터는 이후 챕터의 기반이 됩니다.

단계 선택

Phase 1

코드란 무엇인가?

코드란 무엇이며, 컴퓨터는 코드를 어떻게 실행하는지 이해합니다. AI가 생성한 코드를 읽고, 평가하고, 방향을 제시할 수 있는 기초를 다집니다. 코드를 읽을 수 없으면 AI를 감독할 수 없습니다.

챕터 01–04 · 기초
Phase 2

Python 작성하기

Python 실전 활용 능력을 키웁니다: 함수(function), 자료 구조(data structure), 클래스(class), 파일 처리, 테스트. 어떤 함수든 읽을 수 있고, 테스트를 작성하며, AI가 생성한 코드에서 미묘한 버그를 찾아낼 수 있는 수준까지 도달합니다.

챕터 05–09 · Python
Phase 3

개발 도구

터미널(Terminal), Git, GitHub. Claude Code가 실제로 사용하는 전문 개발 도구들입니다. AI를 제대로 감독하려면 이 도구들을 먼저 이해해야 합니다.

챕터 10–12 · 개발 도구
Phase 4

구축과 배포

API, Flask, 데이터베이스(Database), Docker, CI/CD, 그리고 전문적인 프로젝트 구조를 배웁니다. 소프트웨어를 단순히 사용하는 단계에서 직접 만들고 세상에 배포하는 단계로 나아갑니다.

챕터 13–18 · 구축과 배포
Phase 5

자료 구조와 복잡도

데이터가 메모리에 존재하는 방식을 이해합니다: 배열(array), 연결 리스트(linked list), 해시 테이블(hash table), 트리(tree), 그래프(graph). Big O 표기법을 배우고, 상황에 맞는 자료 구조를 선택하는 안목을 기릅니다.

챕터 19–23 · 자료 구조와 복잡도
Phase 6

알고리즘

탐색(search), 정렬(sort), 재귀(recursion), 그래프 순회(graph traversal). AI가 생성한 코드가 정말 효율적인지 스스로 판단할 수 있는 알고리즘적 사고력을 기릅니다.

챕터 24–27 · 알고리즘
Phase 7

엔지니어링 기술

디자인 패턴(Design Pattern), 데이터베이스 심화, 고급 테스팅, 코드 아키텍처. 취미 개발자와 전문 엔지니어의 차이를 만드는 실전 역량을 다집니다.

챕터 28–33 · 엔지니어링 기술
Phase 8

시스템과 확장

시스템 설계(System Design), 분산 아키텍처, 네트워킹, 성능 최적화. AI 에이전트가 실제로 동작하는 시스템의 구조를 깊이 이해합니다.

챕터 34–37 · 시스템과 확장
Phase 9

AI 지원 개발 (레벨 1~5)

AI 출력물을 평가하는 것에서 시작하여 커스텀 스킬과 MCP를 구축하는 단계까지 나아갑니다. "AI 코드를 판단할 수 있다"에서 "AI에게 내가 직접 설계한 기능을 부여할 수 있다"로 성장합니다.

챕터 38–42 · AI 지원 개발
Phase 10

AI 오케스트레이션 (레벨 6~8)

하네스 엔지니어링(Harness Engineering), 백그라운드 에이전트, Docker 환경에서의 Claude Code, 자율 에이전트 팀 구성까지. 프로젝트가 성장함에 따라 언제든 다시 돌아와서 참고하세요.

챕터 43–47 · AI 오케스트레이션

부록

용어집, 프롬프트 라이브러리, 자기 평가 퀴즈. 모든 것이 한 곳에 모여 있습니다.

용어집 · 프롬프트 라이브러리 · 자기 평가
Phase 1~8의 모든 챕터는 Phase 9~10을 위한 필수 기초입니다. 함수(function)를 배우는 이유는 Claude Code가 200줄짜리 함수를 생성했을 때, 그것이 의도대로 작동하는지 직접 판단할 수 있어야 하기 때문입니다. 자료 구조와 알고리즘(Phase 5~6)을 배우는 이유는 AI가 만든 코드가 진짜 효율적인지 검증할 수 있어야 하기 때문입니다. 엔지니어링 기술과 시스템 사고(Phase 7~8)를 익히는 이유는 에이전트가 실제 운영 환경에서 동작하기 때문입니다. Git, Docker, API를 다루는 이유는 Claude Code가 브랜치를 만들고, 컨테이너 안에서 실행되며, 여러 서비스를 조율하기 때문입니다. 이 커리큘럼에 우회로는 없습니다—모든 챕터가 다음 단계의 선수 과목입니다.
